分页

分页（Paging）是一种非连续的内存管理方式，它将程序的逻辑地址空间和物理内存空间都划分为固定大小的块，分别称为页面（Page）和页框（Page Frame）。分页技术解决了连续分配方式中的外部碎片问题，提高了内存利用率。

分页的基本概念

分页是一种将程序的逻辑地址空间划分为固定大小的页面，并将这些页面映射到物理内存中的页框的技术。页面和页框的大小通常是2的幂次方，如4KB、8KB等。

分页的主要特点：

  • 非连续分配：程序的页面可以分散存储在物理内存的不同位置
  • 固定大小：页面和页框的大小是固定的
  • 透明性：分页对程序员是透明的，程序员不需要关心分页的实现细节

分页的实现机制

页面和页框

  • 页面（Page）：程序逻辑地址空间被划分为固定大小的块，称为页面
  • 页框（Page Frame）：物理内存被划分为固定大小的块，称为页框
  • 页面大小：通常是2的幂次方，如4KB、8KB等

逻辑地址结构

在分页系统中，逻辑地址由两部分组成：

  • 页号（Page Number）：表示逻辑地址所在的页面
  • 页内偏移（Page Offset）：表示逻辑地址在页面内的偏移量

例如，如果页面大小为4KB（2^12字节），则逻辑地址的低12位表示页内偏移，剩余的高位表示页号。

页表（Page Table）

页表是用于实现逻辑地址到物理地址映射的数据结构。每个进程都有一个页表，页表中存储了每个页面对应的页框号。

  • 页表项（Page Table Entry，PTE）：页表中的每一项，包含了页框号和其他控制信息
  • 页表基址寄存器：存储页表的起始地址

地址转换过程

  1. CPU生成逻辑地址
  2. 从逻辑地址中提取页号和页内偏移
  3. 使用页号作为索引，在页表中查找对应的页框号
  4. 将页框号和页内偏移组合，得到物理地址

分页的硬件支持

内存管理单元（MMU）

内存管理单元（Memory Management Unit，MMU）是实现分页的硬件部件，负责逻辑地址到物理地址的转换。

快表（TLB）

快表（Translation Lookaside Buffer，TLB）是一种高速缓存，用于存储最近使用的页表项，以加速地址转换过程。

  • TLB命中：如果所需的页表项在TLB中，则直接使用，无需访问内存中的页表
  • TLB未命中：如果所需的页表项不在TLB中，则需要访问内存中的页表，并将结果存入TLB

分页的优缺点

优点

  • 无外部碎片：分页消除了外部碎片问题
  • 内存利用率高：可以更有效地利用内存空间
  • 程序大小灵活：程序可以大于物理内存大小（结合虚拟内存技术）
  • 共享和保护：可以实现页面的共享和保护

缺点

  • 内部碎片：每个页面的最后一页可能产生内部碎片
  • 页表开销：页表需要占用额外的内存空间
  • 地址转换开销：需要访问页表进行地址转换

多级页表

随着地址空间的增大，单级页表会变得非常大。多级页表通过将页表本身也进行分页，解决了这个问题。

两级页表

两级页表将页表分为页目录和页表两级：

  • 页目录：存储页表的地址
  • 页表：存储页面到页框的映射关系

地址转换过程：

  1. 从逻辑地址中提取页目录索引、页表索引和页内偏移
  2. 使用页目录索引在页目录中查找对应的页表地址
  3. 使用页表索引在页表中查找对应的页框号
  4. 将页框号和页内偏移组合，得到物理地址

多级页表的优点

  • 节省内存空间：不需要为整个地址空间分配页表
  • 支持大地址空间：可以支持更大的逻辑地址空间
  • 灵活的内存管理：可以根据需要动态分配页表

倒排页表

倒排页表（Inverted Page Table）是一种特殊的页表结构，它使用页框号作为索引，而不是页号。

倒排页表的特点

  • 页表大小固定：页表大小等于物理内存中的页框数量
  • 节省空间：对于大地址空间，倒排页表比传统页表更节省空间
  • 查找复杂：查找过程比传统页表复杂

倒排页表的实现

倒排页表通常需要结合哈希表来实现快速查找：

  1. 使用进程ID和页号作为哈希键
  2. 在哈希表中查找对应的页表项
  3. 从页表项中获取页框号

分页与分段的区别

比较项 分页 分段
划分单位 固定大小的页面 可变大小的段
地址空间 一维地址空间 二维地址空间
碎片问题 内部碎片 外部碎片
共享和保护 页面级共享和保护 段级共享和保护
程序员可见性 对程序员透明 对程序员可见

分页的应用场景

  • 通用操作系统：如Windows、Linux等
  • 虚拟内存系统：结合虚拟内存技术，实现大程序在小内存中运行
  • 内存保护：实现进程间的内存隔离
  • 内存共享：实现进程间的内存共享
