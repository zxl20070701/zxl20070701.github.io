分段

分段（Segmentation）是一种将程序的逻辑地址空间划分为若干个逻辑段的内存管理方式。每个段代表程序的一个逻辑单元，如代码段、数据段、堆栈段等。分段技术更好地反映了程序的逻辑结构，便于程序的共享和保护。

分段的基本概念

分段是一种将程序的逻辑地址空间划分为若干个可变大小的逻辑段的技术。每个段代表程序的一个逻辑单元，具有独立的逻辑意义。

分段的主要特点：

  • 逻辑划分：按照程序的逻辑结构进行划分
  • 可变大小：每个段的大小可以根据需要动态变化
  • 二维地址空间：逻辑地址由段号和段内偏移组成
  • 程序员可见：分段对程序员是可见的

分段的实现机制

逻辑地址结构

在分段系统中，逻辑地址由两部分组成：

  • 段号（Segment Number）：表示逻辑地址所在的段
  • 段内偏移（Segment Offset）：表示逻辑地址在段内的偏移量

段表（Segment Table）

段表是用于实现逻辑地址到物理地址映射的数据结构。每个进程都有一个段表，段表中存储了每个段的起始地址（基址）和长度（界限）。

  • 段表项（Segment Table Entry）：段表中的每一项，包含了段的基址、界限和其他控制信息
  • 段表基址寄存器：存储段表的起始地址

地址转换过程

  1. CPU生成逻辑地址
  2. 从逻辑地址中提取段号和段内偏移
  3. 使用段号作为索引，在段表中查找对应的段表项
  4. 检查段内偏移是否超出段的界限
  5. 如果未超出，则将段的基址与段内偏移相加，得到物理地址
  6. 如果超出，则产生地址越界中断

分段的优缺点

优点

  • 逻辑清晰：分段反映了程序的逻辑结构
  • 便于共享：可以实现段的共享
  • 便于保护：可以对不同的段设置不同的保护属性
  • 动态增长：段的大小可以动态增长

缺点

  • 外部碎片：分段会产生外部碎片
  • 地址转换开销：需要访问段表进行地址转换
  • 段表开销：段表需要占用额外的内存空间

分段与分页的比较

比较项 分段 分页
划分单位 可变大小的段 固定大小的页面
地址空间 二维地址空间 一维地址空间
碎片问题 外部碎片 内部碎片
共享和保护 段级共享和保护 页面级共享和保护
程序员可见性 对程序员可见 对程序员透明
逻辑结构 反映程序的逻辑结构 不反映程序的逻辑结构

分段的应用场景

  • 早期操作系统：如MULTICS、Intel 80286等
  • 需要逻辑划分的系统：如代码、数据、堆栈分离的系统
  • 需要段级共享和保护的系统：如多用户系统

现代操作系统中的分段

在现代操作系统中，分段通常与分页结合使用，形成段页式内存管理：

  • Intel x86架构：支持分段和分页，但现代操作系统通常只使用分页
  • Linux：使用分页，分段主要用于内核和用户空间的隔离
  • Windows：使用分页，分段主要用于兼容性支持

分段的发展趋势

随着计算机技术的发展，分段技术逐渐被分页技术所取代，主要原因包括：

  • 外部碎片问题：分段会产生外部碎片，影响内存利用率
  • 实现复杂性：分段的实现比分页复杂
  • 性能考虑：分页的地址转换比分段更高效

但在某些特定场景下，分段仍然有其应用价值，特别是在需要逻辑划分和段级共享的系统中。
