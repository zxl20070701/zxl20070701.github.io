连续分配

连续分配（Continuous Allocation）是内存管理的一种基本方式，它将整个程序加载到内存中的一个连续区域。这种分配方式简单直观，是早期操作系统中常用的内存管理方法。

连续分配的基本概念

连续分配是指为一个程序或进程分配一块连续的内存空间，程序的指令和数据都存储在这块连续的空间中。这种方式下，程序的逻辑地址空间与物理地址空间是连续的。

连续分配的主要特点是：

  • 内存空间连续性：程序占用的内存空间是连续的
  • 地址转换简单：逻辑地址可以直接映射到物理地址
  • 实现简单：硬件支持和操作系统实现都相对简单

连续分配的实现方式

单一连续分配

单一连续分配是最简单的内存管理方式，整个内存空间只分配给一个程序使用。这种方式下，内存中只有一个程序在运行。

特点

  • 内存利用率高：整个内存空间都分配给一个程序
  • 实现简单：不需要复杂的内存管理算法
  • 并发性差：一次只能运行一个程序

适用场景

早期单道批处理系统、嵌入式系统等对并发性要求不高的场景。

固定分区分配

固定分区分配将内存空间划分为若干个固定大小的分区，每个分区可以装入一个程序。分区的大小可以相等，也可以不等。

特点

  • 分区大小固定：分区大小在系统初始化时确定，运行期间不变
  • 并发性好：可以同时运行多个程序
  • 内存碎片：会产生内部碎片

分区分配算法

  • 首次适应（First Fit）：从内存的起始位置开始查找，选择第一个足够大的空闲分区
  • 最佳适应（Best Fit）：查找所有空闲分区，选择大小最接近程序需求的分区
  • 最差适应（Worst Fit）：查找所有空闲分区，选择最大的空闲分区

动态分区分配

动态分区分配根据程序的实际需求动态地分配内存空间，分区的大小和数量都是可变的。

特点

  • 分区大小可变：根据程序需求动态分配
  • 内存利用率高：减少了内部碎片
  • 外部碎片：会产生外部碎片

分区分配算法

  • 首次适应（First Fit）：从内存的起始位置开始查找，选择第一个足够大的空闲分区
  • 最佳适应（Best Fit）：查找所有空闲分区，选择大小最接近程序需求的分区
  • 最差适应（Worst Fit）：查找所有空闲分区，选择最大的空闲分区
  • 邻近适应（Next Fit）：从上次分配的位置开始查找，选择第一个足够大的空闲分区

连续分配的地址转换

在连续分配方式下，逻辑地址到物理地址的转换相对简单：

  • 基址寄存器：存储程序在内存中的起始地址
  • 界限寄存器：存储程序的长度或结束地址
  • 地址转换：物理地址 = 逻辑地址 + 基址寄存器的值

地址转换过程：

  1. CPU生成逻辑地址
  2. 检查逻辑地址是否超出界限寄存器的值
  3. 如果未超出，则将逻辑地址与基址寄存器的值相加，得到物理地址
  4. 如果超出，则产生地址越界中断

连续分配的内存保护

连续分配方式下，内存保护主要依靠硬件支持：

  • 基址寄存器：限制程序访问的最低内存地址
  • 界限寄存器：限制程序访问的最高内存地址
  • 地址检查：每次内存访问时，硬件都会检查地址是否在合法范围内

连续分配的优缺点

优点

  • 实现简单：硬件和软件实现都相对简单
  • 地址转换快：逻辑地址可以直接映射到物理地址
  • 内存访问效率高：连续的内存空间有利于提高缓存命中率

缺点

  • 内存碎片：会产生内部碎片和外部碎片
  • 内存利用率低：碎片导致内存空间浪费
  • 程序大小受限：程序大小不能超过内存可用空间
  • 缺乏灵活性：程序必须加载到连续的内存空间中

内存碎片问题

内部碎片

内部碎片是指分配给程序的内存空间中，有一部分没有被使用，但无法分配给其他程序使用。例如，在固定分区分配中，如果程序大小小于分区大小，则分区中剩余的空间就是内部碎片。

外部碎片

外部碎片是指内存中分散的小块空闲空间，虽然总大小足够，但由于不连续，无法分配给程序使用。例如，在动态分区分配中，经过多次分配和释放后，内存中会出现许多小块的空闲空间。

碎片整理

为了解决外部碎片问题，可以采用碎片整理技术：

  • 压缩（Compaction）：将内存中的程序移动，使空闲空间连续
  • 分区合并：将相邻的空闲分区合并成一个大的空闲分区

连续分配的应用场景

  • 嵌入式系统：内存资源有限，程序规模较小
  • 实时系统：对内存访问速度要求高
  • 早期操作系统：硬件资源有限，实现简单

连续分配的发展趋势

随着计算机技术的发展，连续分配方式逐渐被更先进的内存管理方式所取代，如分页、分段和虚拟内存等。但在某些特定场景下，连续分配仍然有其应用价值。
