进程创建/终止

进程的创建与终止是操作系统进程管理中的两个重要操作。进程创建是指操作系统为新程序的执行分配资源并建立必要的数据结构，而进程终止是指操作系统回收进程占用的资源并销毁相关数据结构。

进程创建

进程创建是指操作系统为新程序的执行创建一个新的进程。进程的创建通常通过系统调用来实现，如UNIX系统中的fork()系统调用。

进程创建的原因

  • 用户登录：用户登录系统时，系统为用户创建登录进程
  • 作业调度：批处理系统中，作业调度程序为待执行的作业创建进程
  • 提供服务：操作系统为用户请求创建服务进程，如打印服务进程
  • 应用请求：正在运行的应用进程根据自身需要创建子进程

进程创建的过程

  1. 申请空白PCB：为新进程分配一个唯一的进程标识符，并从PCB集合中申请一个空闲的PCB
  2. 为新进程分配资源：为新进程的程序、数据和用户栈分配必要的内存空间
  3. 初始化PCB：初始化PCB中的各种信息，如进程标识符、处理器状态信息、进程控制信息等
  4. 将新进程插入就绪队列：将新进程的PCB插入到就绪队列中，等待调度执行

进程创建的实现方式

  • 父进程创建子进程：父进程通过系统调用创建子进程，子进程继承父进程的部分属性
  • 操作系统创建初始进程：系统启动时，操作系统创建初始进程（如UNIX中的init进程）
  • 批处理作业创建：批处理系统中，作业调度程序为用户作业创建进程

进程创建的相关系统调用

  • fork()：UNIX/Linux系统中创建子进程的系统调用，子进程是父进程的副本
  • exec()：用新程序替换当前进程的代码和数据，通常与fork()配合使用
  • CreateProcess()：Windows系统中创建新进程的系统调用
  • clone()：Linux系统中创建线程的系统调用，可以指定共享的资源

进程终止

进程终止是指操作系统回收进程占用的资源并销毁相关数据结构。进程终止可以是正常终止，也可以是异常终止。

进程终止的原因

  • 正常结束：进程完成了所有任务，执行完最后一条语句后正常结束
  • 异常结束：进程在执行过程中发生错误或异常，如除零错误、非法内存访问等
  • 外界干预：操作系统或用户强制终止进程，如收到KILL信号
  • 父进程终止：父进程终止时，其子进程可能被操作系统自动终止

进程终止的过程

  1. 查找终止进程的PCB：根据进程标识符查找要终止进程的PCB
  2. 终止进程的执行：如果进程正在执行，则立即停止其执行，并设置调度标志为真，以便调度其他进程
  3. 终止子进程：如果该进程有子孙进程，则终止其所有子孙进程
  4. 回收资源：回收进程占用的所有资源，包括内存、文件、I/O设备等
  5. 将进程从队列中移除：将被终止进程的PCB从所在队列中移除
  6. 回收PCB：回收被终止进程的PCB，以便其他进程使用

进程终止的相关系统调用

  • exit()：进程主动终止自身的系统调用，UNIX/Linux系统中使用
  • _exit()：立即终止进程，不执行清理操作的系统调用
  • abort()：异常终止进程的系统调用
  • TerminateProcess()：Windows系统中终止进程的系统调用

进程创建与终止的性能考虑

进程创建与终止虽然是操作系统正常运行的必要机制，但也会带来一定的性能开销：

  • 创建开销：为新进程分配资源、初始化PCB等操作需要时间
  • 终止开销：回收资源、销毁PCB等操作需要时间
  • 上下文切换开销：进程创建和终止涉及上下文切换，会带来额外开销

进程创建与终止的安全性考虑

  • 权限检查：创建进程时需要检查用户权限，防止未授权用户创建进程
  • 资源限制：限制单个用户可以创建的进程数量，防止资源耗尽
  • 僵尸进程处理：及时处理已终止但未被父进程回收的僵尸进程
  • 孤儿进程处理：父进程终止时，妥善处理其子进程，防止产生孤儿进程

进程创建与终止的应用场景

  • Shell命令执行：Shell为每个命令创建一个新进程
  • Web服务器：为每个客户端请求创建一个新进程或线程
  • 数据库系统：为每个用户连接创建一个新进程
  • 批处理系统：为每个作业创建一个新进程
