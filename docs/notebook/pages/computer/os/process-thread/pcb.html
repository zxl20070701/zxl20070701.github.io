控制块（PCB）

进程控制块（Process Control Block，PCB）是操作系统用于管理进程的数据结构，是进程存在的唯一标志。PCB中包含了操作系统所需的所有进程信息，是进程管理和调度的核心数据结构。

PCB的作用

PCB在操作系统中扮演着至关重要的角色：

  • 进程存在的唯一标志：当进程被创建时，系统为其创建PCB；当进程被撤销时，系统回收其PCB
  • 进程管理的依据：操作系统通过PCB对进程进行管理，包括进程调度、资源分配、状态转换等
  • 进程间通信的基础：PCB中包含了进程通信所需的信息，如消息队列、信号量等
  • 中断处理和恢复的依据：当中断发生时，系统通过PCB保存和恢复进程的执行现场

PCB的组成

PCB通常包含以下信息：

进程标识信息

  • 进程标识符（PID，Process ID）：唯一标识一个进程的数字
  • 用户标识符（UID，User ID）：标识进程所属用户的ID
  • 组标识符（GID，Group ID）：标识进程所属用户组的ID
  • 父进程标识符（PPID，Parent Process ID）：创建该进程的父进程的PID

处理器状态信息

处理器状态信息保存了进程执行时的CPU现场，当进程被中断时，这些信息被保存到PCB中；当进程恢复执行时，这些信息被从PCB中恢复到CPU中。

  • 通用寄存器：保存进程执行时的通用寄存器内容
  • 程序计数器（PC，Program Counter）：保存下一条要执行的指令地址
  • 程序状态字（PSW，Program Status Word）：保存处理器的当前状态，如条件码、中断屏蔽位等
  • 栈指针（SP，Stack Pointer）：指向进程用户栈的栈顶

进程控制信息

  • 进程状态：标识进程当前的状态，如就绪、执行、阻塞等
  • 进程优先级：用于进程调度的优先级信息
  • 程序和数据地址：进程的程序和数据在内存中的地址
  • 进程同步和通信机制：如消息队列指针、信号量等
  • 资源清单：进程所需的资源列表，如打开的文件、I/O设备等
  • 链接指针：用于将PCB组织成队列的指针

资源管理信息

  • 内存管理信息：如页表、段表等内存管理数据结构
  • I/O设备信息：进程使用的I/O设备列表
  • 文件信息：进程打开的文件列表及文件描述符
  • 账户信息：记录进程使用的CPU时间、内存等资源的使用情况

PCB的组织方式

操作系统需要将多个进程的PCB组织起来，以便于管理和调度。常见的组织方式有：

线性表方式

将所有PCB组织成一个线性表，适用于系统中进程数量较少的情况。优点是简单直观，缺点是查找效率低。

链接表方式

将相同状态的PCB链接成一个队列，如就绪队列、阻塞队列等。这是最常见的组织方式，便于按照进程状态进行管理。

索引表方式

建立多个索引表，如就绪索引表、阻塞索引表等，每个表项指向一个PCB。这种方式可以快速定位特定状态的进程。

PCB的创建与撤销

PCB的创建

当创建一个新进程时，系统需要执行以下步骤：

  1. 为新进程分配一个唯一的进程标识符（PID）
  2. 为PCB分配内存空间
  3. 初始化PCB的各个字段，如进程状态、优先级等
  4. 将PCB插入到相应的队列中（如就绪队列）

PCB的撤销

当进程终止时，系统需要执行以下步骤：

  1. 从相应队列中移除该进程的PCB
  2. 回收进程占用的所有资源
  3. 回收PCB占用的内存空间
  4. 通知父进程子进程的终止状态

PCB与进程调度

PCB在进程调度中起着关键作用：

  • 调度信息：PCB中包含了进程优先级、调度策略等调度相关信息
  • 上下文保存：进程切换时，当前进程的CPU现场被保存到其PCB中
  • 上下文恢复：新进程被调度执行时，其CPU现场从其PCB中恢复到CPU中
  • 资源管理：PCB中记录了进程的资源使用情况，为调度决策提供依据

PCB与进程通信

PCB在进程通信中也扮演着重要角色：

  • 通信机制：PCB中包含了进程通信所需的数据结构，如消息队列、共享内存等
  • 同步机制：PCB中记录了进程的同步状态，如等待的信号量、互斥锁等
  • 进程关系：PCB中记录了进程的父子关系，为进程间通信提供基础

PCB的实现示例

以下是一个简化的PCB结构示例（以C语言表示）： 

struct PCB {
    int pid;                    // 进程标识符
    int ppid;                   // 父进程标识符
    int uid;                    // 用户标识符
    int gid;                    // 组标识符

    enum proc_state state;      // 进程状态
    int priority;               // 进程优先级

    struct context context;     // CPU现场
    unsigned long *pgdir;       // 页目录地址

    struct file *files[NR_FILES]; // 打开的文件
    struct inode *pwd;          // 当前工作目录

    unsigned long start_time;   // 进程开始时间
    unsigned long cpu_time;     // CPU使用时间

    struct list_head list;      // 链表指针，用于组织PCB
    struct list_head children;  // 子进程链表
    struct PCB *parent;         // 父进程指针
};

PCB的存储位置

PCB通常存储在内核空间中，由操作系统内核管理。用户程序无法直接访问PCB，只能通过系统调用来间接操作PCB中的信息。

PCB与系统调用

许多系统调用都会涉及PCB的操作：

  • fork()：创建新进程时，为新进程创建PCB
  • exec()：加载新程序时，更新PCB中的程序信息
  • wait()：等待子进程结束时，访问子进程的PCB
  • getpid()：获取当前进程的PID，从当前进程的PCB中读取
