进程与线程

进程的基本概念

进程（Process） 是程序的一次执行过程，是系统进行资源分配和调度的基本单位。每个进程都有独立的内存空间、数据栈以及其他用于跟踪进程执行的辅助数据。

进程的状态

进程在其生命周期中会经历不同的状态：

  • 创建（New）：进程正在被创建
  • 就绪（Ready）：进程已准备好，等待分配CPU
  • 运行（Running）：进程正在CPU上执行
  • 阻塞（Blocked）：进程等待某个事件（如I/O完成）
  • 终止（Terminated）：进程执行完毕

线程的基本概念

线程（Thread） 是进程内的一个执行单元，是CPU调度和分派的基本单位。一个进程可以包含多个线程，这些线程共享进程的资源，但有自己的栈空间和程序计数器。

进程与线程的区别

进程和线程的主要区别：

  • 资源拥有：进程是资源分配的基本单位，线程不拥有系统资源
  • 独立性：进程间相互独立，线程间共享进程资源
  • 开销：创建和销毁进程的开销远大于线程
  • 通信：进程间通信需要特殊机制，线程间可以直接通信
  • 健壮性：一个进程崩溃不会影响其他进程，一个线程崩溃可能导致整个进程崩溃

进程调度算法

操作系统使用不同的算法来决定哪个进程获得CPU时间：

  • 先来先服务（FCFS）：按照进程到达的顺序进行调度
  • 短作业优先（SJF）：优先调度执行时间短的进程
  • 时间片轮转（RR）：每个进程分配一个时间片，轮流执行
  • 优先级调度：根据进程的优先级进行调度
  • 多级反馈队列：结合时间片轮转和优先级调度的优点

进程间通信（IPC）

进程间通信是指在不同进程之间传递信息或信号的机制：

  • 管道（Pipe）：单向通信，用于父子进程间通信
  • 命名管道（Named Pipe）：可以在无亲缘关系的进程间通信
  • 消息队列（Message Queue）：存储在内核中的消息链表
  • 共享内存（Shared Memory）：多个进程共享同一块内存区域
  • 信号量（Semaphore）：用于进程间同步和互斥
  • 套接字（Socket）：可用于不同主机间的进程通信

线程同步机制

为了防止多个线程同时访问共享资源造成冲突，需要使用同步机制：

  • 互斥锁（Mutex）：确保同一时间只有一个线程访问共享资源
  • 读写锁（Read-Write Lock）：允许多个读线程同时访问，但写线程独占
  • 条件变量（Condition Variable）：线程可以等待特定条件满足
  • 信号量（Semaphore）：控制同时访问共享资源的线程数量
  • 自旋锁（Spinlock）：线程循环等待锁的释放
