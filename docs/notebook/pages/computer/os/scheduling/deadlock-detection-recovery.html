检测与恢复

死锁检测与恢复（Deadlock Detection and Recovery）是一种处理死锁的策略，它允许死锁发生，但会定期检测系统中是否存在死锁，并在检测到死锁时采取措施恢复系统。与死锁预防和避免不同，死锁检测与恢复是一种更为灵活的策略，适用于那些死锁发生概率较低或者死锁影响较小的系统。

死锁检测与恢复的基本思想

死锁检测与恢复的基本思想是：允许系统进入死锁状态，但会定期检测系统中是否存在死锁。如果检测到死锁，系统会采取措施打破死锁，使系统恢复正常运行。这种策略的优点是系统开销小，资源利用率高；缺点是死锁发生时可能会影响系统性能。

死锁检测

检测时机

  • 周期性检测：系统定期（如每隔一段时间）检测是否存在死锁
  • 资源请求时检测：每当进程请求资源时，系统检测是否存在死锁
  • 资源释放时检测：每当进程释放资源时，系统检测是否存在死锁

检测方法

资源分配图法

资源分配图法是一种基于图的死锁检测方法。如果资源分配图中存在环路，且环路中的每个资源节点都只有一个实例，则系统处于死锁状态。

检测步骤：

  1. 构建资源分配图
  2. 检测图中是否存在环路
  3. 如果存在环路且环路中的每个资源节点都只有一个实例，则系统处于死锁状态

死锁检测算法

死锁检测算法是一种基于矩阵的死锁检测方法，类似于银行家算法中的安全性算法。

数据结构：

  • Available：可利用资源向量
  • Allocation：分配矩阵
  • Request：请求矩阵

检测步骤：

  1. 初始化：令Work = Available，Finish[i] = false（对于所有i）
  2. 查找：查找满足以下条件的进程i：
    • Finish[i] = false
    • Request[i] ≤ Work
  3. 如果找到这样的进程，则执行以下操作：
    • Work = Work + Allocation[i]
    • Finish[i] = true
    • 转到步骤2
  4. 如果存在进程的Finish[i] = false，则这些进程处于死锁状态

死锁恢复

当检测到死锁时，系统需要采取措施打破死锁，使系统恢复正常运行。常见的死锁恢复方法包括：

进程终止

终止一个或多个死锁进程，释放它们占有的资源，从而打破死锁。

终止策略

  • 终止所有死锁进程：简单但代价大
  • 逐个终止死锁进程：每次终止一个进程，然后重新检测是否仍存在死锁

选择终止进程的准则

  • 进程优先级：优先终止优先级低的进程
  • 进程运行时间：优先终止运行时间短的进程
  • 进程资源需求：优先终止资源需求少的进程
  • 进程类型：优先终止批处理进程，保留交互式进程

资源抢占

抢占一个或多个死锁进程占有的资源，分配给其他进程，从而打破死锁。

选择抢占资源的准则

  • 资源类型：优先抢占易于恢复的资源
  • 进程影响：优先抢占对进程影响小的资源
  • 资源数量：优先抢占数量多的资源

资源抢占的步骤

  1. 选择一个牺牲品进程
  2. 回滚该进程到某个安全状态
  3. 将该进程占有的资源分配给其他进程
  4. 重新启动该进程

进程回滚

将死锁进程回滚到某个安全状态，释放它们占有的资源，从而打破死锁。

回滚策略

  • 完全回滚：将进程回滚到初始状态
  • 部分回滚：将进程回滚到某个中间状态

回滚点的选择

  • 检查点：将进程回滚到最近的检查点
  • 资源分配点：将进程回滚到最近获得资源的点

人工干预

在某些情况下，系统可能会通知操作员或用户，由他们来决定如何处理死锁。

死锁恢复策略的选择

选择合适的死锁恢复策略需要考虑以下因素：

  • 死锁进程的重要性：优先保护重要进程
  • 死锁进程的运行时间：优先保护运行时间长的进程
  • 死锁进程的资源需求：优先保护资源需求多的进程
  • 系统性能影响：选择对系统性能影响最小的恢复策略

死锁检测与恢复的优缺点

优点

  • 系统开销小：相比死锁预防和避免，系统开销较小
  • 资源利用率高：允许系统最大限度地利用资源
  • 灵活性强：可以根据系统状态动态调整策略

缺点

  • 死锁发生时系统性能下降：死锁进程无法继续执行
  • 恢复过程复杂：需要选择合适的恢复策略
  • 可能导致数据不一致：进程终止或回滚可能导致数据不一致

死锁检测与恢复的应用实例

操作系统

大多数现代操作系统采用死锁检测与恢复策略来处理死锁。例如，Windows和Linux都使用超时机制来检测死锁，并在检测到死锁时终止相关进程。

数据库系统

数据库系统通常采用死锁检测与恢复策略来处理事务死锁。系统会定期检测是否存在死锁，并在检测到死锁时回滚一个或多个事务。

分布式系统

分布式系统通常采用超时机制和协调器来检测死锁。如果某个进程在一定时间内无法获得所需资源，协调器会检测到死锁并采取措施恢复系统。
