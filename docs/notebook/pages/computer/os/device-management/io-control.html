I/O控制方式

I/O控制方式是操作系统管理I/O设备的基本方法，它决定了CPU与I/O设备之间如何交换数据。随着计算机技术的发展，I/O控制方式经历了从简单到复杂的演进过程，主要包括程序查询、中断、DMA和通道四种方式。

程序查询方式（Programmed I/O）

程序查询方式是最简单的I/O控制方式，CPU通过不断查询I/O设备的状态来控制数据传输。

工作原理

  1. CPU向I/O设备发送命令
  2. CPU不断查询I/O设备的状态寄存器
  3. 当设备准备好时，CPU执行数据传输
  4. 传输完成后，CPU继续执行后续程序

特点

  • 简单直观：实现简单，易于理解
  • CPU效率低：CPU需要不断查询设备状态，浪费大量CPU时间
  • 实时性差：CPU无法及时响应设备请求

适用场景

简单的、低速的I/O设备，如早期的打印机、键盘等。

中断方式（Interrupt-driven I/O）

中断方式是一种异步的I/O控制方式，当I/O设备准备好时，向CPU发送中断信号，CPU暂停当前程序，转而执行中断处理程序。

工作原理

  1. CPU向I/O设备发送命令
  2. CPU继续执行其他程序
  3. 当设备准备好时，向CPU发送中断信号
  4. CPU响应中断，执行中断处理程序
  5. 中断处理程序完成数据传输
  6. CPU恢复执行被中断的程序

特点

  • CPU效率高：CPU不需要不断查询设备状态
  • 实时性好：CPU能够及时响应设备请求
  • 实现复杂：需要中断控制器和中断处理机制

中断类型

  • 可屏蔽中断：CPU可以选择是否响应
  • 不可屏蔽中断：CPU必须立即响应
  • 向量中断：中断源提供中断向量
  • 非向量中断：中断源不提供中断向量

适用场景

中速的I/O设备，如磁盘、网络接口等。

DMA方式（Direct Memory Access）

DMA方式是一种高速的I/O控制方式，它允许I/O设备直接与内存交换数据，无需CPU干预。

工作原理

  1. CPU向DMA控制器发送命令，包括数据传输方向、内存地址、数据长度等
  2. CPU继续执行其他程序
  3. DMA控制器控制I/O设备与内存直接交换数据
  4. 数据传输完成后，DMA控制器向CPU发送中断信号
  5. CPU响应中断，处理传输完成后的工作

特点

  • 高速传输：数据传输速度快，无需CPU干预
  • CPU效率高：CPU可以同时执行其他程序
  • 硬件复杂：需要DMA控制器和复杂的控制逻辑

DMA传输方式

  • 突发传输：一次传输大量数据
  • 周期窃取：在每个总线周期窃取一个总线周期进行传输
  • 透明传输：在CPU不使用总线时进行传输

适用场景

高速的I/O设备，如磁盘、图形卡、网络接口等。

通道方式（Channel I/O）

通道方式是大型计算机系统中使用的一种高级I/O控制方式，它使用专门的I/O处理器（通道）来管理I/O设备。

工作原理

  1. CPU向通道发送I/O命令，包括通道程序（一组通道指令）
  2. CPU继续执行其他程序
  3. 通道解释执行通道程序，控制I/O设备工作
  4. 通道完成I/O操作后，向CPU发送中断信号
  5. CPU响应中断，处理I/O完成后的工作

特点

  • 高度并行：CPU、通道和I/O设备可以并行工作
  • 功能强大：通道可以执行复杂的I/O操作
  • 成本高昂：需要专门的通道处理器和复杂的控制逻辑

通道类型

  • 选择通道：一次只能连接一个设备，但传输速率高
  • 数组多路通道：可以连接多个设备，按数组方式传输数据
  • 字节多路通道：可以连接多个设备，按字节方式传输数据

适用场景

大型计算机系统，如IBM大型机。

I/O控制方式的比较

控制方式 CPU效率 实现复杂度 传输速度 适用场景
程序查询 低 简单 慢 简单设备
中断 中 中等 中 中速设备
DMA 高 复杂 快 高速设备
通道 最高 最复杂 最快 大型系统

发展趋势

随着计算机技术的发展，I/O控制方式也在不断演进：

  • 从简单到复杂：从程序查询到通道方式
  • 从低效到高效：CPU效率不断提高
  • 从专用到通用：I/O控制方式越来越通用化
  • 从硬件到软件：软件在I/O控制中的作用越来越大
