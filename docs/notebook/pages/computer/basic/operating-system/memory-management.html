内存管理

内存管理的基本概念

内存管理（Memory Management） 是操作系统的重要功能之一，负责管理计算机的主存储器，包括内存的分配、回收、保护和扩充等。

内存分配方式

操作系统采用不同的内存分配方式：

  • 连续分配：为每个进程分配连续的内存空间
    • 单一连续分配：整个内存只分配给一个进程
    • 固定分区分配：将内存划分为固定大小的分区
    • 动态分区分配：根据进程需求动态划分内存
  • 非连续分配：允许进程使用不连续的内存空间
    • 分页存储管理：将内存划分为固定大小的页面
    • 分段存储管理：将内存按逻辑单元划分为段
    • 段页式存储管理：结合分段和分页的优点

分页存储管理

分页存储管理将内存划分为固定大小的页面（Page），同时将进程的逻辑地址空间也划分为相同大小的页：

  • 页面（Page）：逻辑地址空间中的固定大小块
  • 页框（Frame）：物理内存中的固定大小块
  • 页表（Page Table）：记录逻辑页面到物理页框的映射关系
  • 地址转换：通过页表将逻辑地址转换为物理地址

分段存储管理

分段存储管理按照程序的逻辑结构将地址空间划分为若干段：

  • 代码段：存放程序的可执行代码
  • 数据段：存放程序的全局变量和静态变量
  • 堆段：存放动态分配的内存
  • 栈段：存放函数调用的参数、返回地址和局部变量

虚拟内存

虚拟内存（Virtual Memory） 技术允许程序使用比实际物理内存更大的地址空间：

  • 请求分页：只在需要时才将页面调入内存
  • 页面置换：当内存不足时，选择页面换出到磁盘
  • 页面置换算法：决定哪些页面应该被换出
    • 最佳置换算法（OPT）
    • 先进先出算法（FIFO）
    • 最近最久未使用算法（LRU）
    • 时钟置换算法（Clock）

内存保护

内存保护机制确保进程只能访问自己的内存空间，防止相互干扰：

  • 界地址保护：设置内存的上界和下界寄存器
  • 存储键保护：为每个存储块分配保护键
  • 环保护：将系统分为不同特权级别的环

内存碎片

内存碎片是指内存中无法被利用的小块空闲空间：

  • 外部碎片：分区间的碎片，可以通过紧凑技术解决
  • 内部碎片：分区内的碎片，分配给进程但未使用
  • 紧凑技术：通过移动内存中的进程来合并碎片
