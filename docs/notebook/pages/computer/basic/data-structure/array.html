数组与链表

数组的基本概念

数组（Array） 是一种线性数据结构，它用一组连续的内存空间来存储一组具有相同类型的数据。数组中的每个元素可以通过索引快速访问。

数组的特点

  • 随机访问：可以通过索引在O(1)时间内访问任意元素
  • 内存连续：元素在内存中连续存储
  • 固定大小：传统数组的大小在创建时确定，不能动态改变
  • 类型一致：所有元素必须是相同的数据类型

数组的优缺点

优点：

  • 访问速度快，支持随机访问
  • 内存局部性好，缓存命中率高
  • 实现简单，使用方便

缺点：

  • 大小固定，插入和删除操作效率低
  • 需要预先知道数据规模
  • 可能造成内存浪费或溢出

链表的基本概念

链表（Linked List） 是一种线性数据结构，它通过指针将一组零散的内存块串联起来。每个内存块称为节点，包含数据域和指针域。

链表的类型

  • 单向链表：每个节点只有一个指向下一个节点的指针
  • 双向链表：每个节点有两个指针，分别指向前一个和后一个节点
  • 循环链表：最后一个节点的指针指向第一个节点，形成环状

链表的特点

  • 动态大小：可以根据需要动态增加或减少节点
  • 内存分散：节点在内存中不需要连续存储
  • 插入删除高效：在已知位置插入或删除节点只需要O(1)时间
  • 顺序访问：访问特定位置的元素需要从头开始遍历

链表的优缺点

优点：

  • 大小动态可变，内存利用率高
  • 插入和删除操作效率高
  • 不需要预先分配固定大小的内存

缺点：

  • 不支持随机访问，查找效率低
  • 需要额外的指针空间，内存开销大
  • 内存局部性差，缓存命中率低

数组与链表的比较

比较维度 数组 链表
内存分配 静态分配，连续内存 动态分配，分散内存
访问方式 随机访问，O(1) 顺序访问，O(n)
插入/删除 O(n)，需要移动元素 O(1)，只需修改指针
内存利用率 可能浪费或不足 按需分配，利用率高
缓存友好性 好，连续存储 差，分散存储

应用场景

数组适用于：

  • 数据规模已知且固定
  • 频繁随机访问元素
  • 对内存局部性要求高

链表适用于：

  • 数据规模动态变化
  • 频繁插入和删除操作
  • 内存空间有限且分散
