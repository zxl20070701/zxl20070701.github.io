TCP/IP 协议

TCP/IP（Transmission Control Protocol/Internet Protocol，传输控制协议/网际协议）是一组用于实现网络互连的通信协议，是互联网的基础协议。

协议层次结构

TCP/IP 协议族通常被认为是一个四层协议系统：

  1. 应用层：为应用程序提供服务（HTTP、FTP、SMTP等）
  2. 传输层：提供端到端的通信（TCP、UDP）
  3. 网络层：负责数据包的路由和转发（IP、ICMP、ARP等）
  4. 网络接口层：处理与物理网络的接口（以太网、WiFi等）

TCP 协议特点

TCP 是一种面向连接的、可靠的传输层协议，具有以下特点：

  • 面向连接：通信前需要建立连接
  • 可靠传输：通过确认机制、重传机制保证数据可靠传输
  • 流量控制：通过滑动窗口机制控制发送速率
  • 拥塞控制：通过拥塞窗口避免网络拥塞
  • 全双工通信：支持双向同时传输

IP 协议特点

IP 是无连接的、不可靠的网络层协议，具有以下特点：

  • 无连接：不建立连接，每个数据包独立路由
  • 不可靠：不保证数据包一定到达，也不保证按顺序到达
  • 分组交换：将数据分割成数据包进行传输
  • 路由选择：根据路由表选择最佳路径

TCP 三次握手

TCP 建立连接的过程称为三次握手：

  1. SYN：客户端发送 SYN 包请求建立连接
  2. SYN-ACK：服务器回应 SYN-ACK 包表示同意建立连接
  3. ACK：客户端发送 ACK 包确认连接建立

TCP 四次挥手

TCP 关闭连接的过程称为四次挥手：

  1. FIN：主动关闭方发送 FIN 包请求关闭连接
  2. ACK：被动方回应 ACK 包确认收到关闭请求
  3. FIN：被动方发送 FIN 包请求关闭连接
  4. ACK：主动方回应 ACK 包确认连接关闭

常见端口号

常见的网络服务及其对应的端口号：

  • 21：FTP 文件传输协议
  • 22：SSH 安全外壳协议
  • 23：Telnet 远程登录协议
  • 25：SMTP 简单邮件传输协议
  • 53：DNS 域名系统
  • 80：HTTP 超文本传输协议
  • 110：POP3 邮局协议版本3
  • 143：IMAP 互联网消息访问协议
  • 443：HTTPS 安全超文本传输协议
