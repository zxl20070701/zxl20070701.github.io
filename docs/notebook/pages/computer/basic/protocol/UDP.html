UDP 协议

UDP（User Datagram Protocol，用户数据报协议）是一种无连接的传输层协议，提供面向事务的简单不可靠信息传送服务。

协议特点

UDP 具有以下主要特点：

  • 无连接：发送数据前不需要建立连接
  • 不可靠：不保证数据报一定到达，也不保证按顺序到达
  • 面向报文：对应用层交下来的报文不合并、不拆分
  • 无拥塞控制：网络拥塞时不会降低发送速率
  • 支持一对一、一对多、多对一和多对多交互通信
  • 首部开销小：只有8个字节，比 TCP 的20个字节要小

UDP 报文结构

UDP 报文由首部和数据两部分组成，首部包含以下字段：

  • 源端口号（16位）
  • 目的端口号（16位）
  • 长度（16位）：UDP 用户数据报的长度
  • 检验和（16位）：用于差错检测
  • 数据：应用层数据

与 TCP 的对比

UDP 与 TCP 的主要区别如下：

特性 UDP TCP
连接 无连接 面向连接
可靠性 不可靠 可靠
传输速度 快 相对较慢
首部开销 8字节 20字节
流量控制 无 有
拥塞控制 无 有

常见应用场景

UDP 适用于以下场景：

  • DNS 查询：域名解析
  • 视频/音频流媒体：实时性要求高，可容忍少量丢包
  • 在线游戏：实时交互，需要低延迟
  • VoIP 语音通话：实时语音传输
  • TFTP 文件传输：简单文件传输协议
  • SNMP 网络管理：简单网络管理协议
  • DHCP 动态主机配置：IP 地址分配

优缺点

优点

  • 传输速度快
  • 首部开销小
  • 无连接建立和释放过程
  • 支持广播和多播

缺点

  • 不可靠传输
  • 可能出现丢包、重复、乱序
  • 无流量控制和拥塞控制
