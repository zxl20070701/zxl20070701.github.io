ICMP 协议

ICMP（Internet Control Message Protocol，互联网控制消息协议）是 TCP/IP 协议族的一个子协议，用于在 IP 主机、路由器之间传递控制消息。

ICMP 协议主要用于网络诊断和错误报告，不传输用户数据。它封装在 IP 数据报中，使用 IP 协议进行传输，协议号为 1。

ICMP 报文类型

ICMP 报文分为两大类：

  • 差错报告报文：报告网络错误
  • 询问报文：用于网络诊断

常见的 ICMP 报文类型

类型 代码 描述
0 0 Echo Reply（回显应答）
3 0-15 Destination Unreachable（目的不可达）
4 0 Source Quench（源站抑制）
5 0-3 Redirect（重定向）
8 0 Echo Request（回显请求）
11 0-1 Time Exceeded（超时）
12 0-1 Parameter Problem（参数问题）
13 0 Timestamp Request（时间戳请求）
14 0 Timestamp Reply（时间戳应答）

ICMP 报文格式

ICMP 报文包含以下字段：

  • 类型（8 位）：指明 ICMP 报文的类型
  • 代码（8 位）：进一步描述 ICMP 报文
  • 校验和（16 位）：检验整个 ICMP 报文
  • 数据部分：根据类型和代码的不同而不同

常见 ICMP 应用

Ping

Ping 程序使用 ICMP Echo Request 和 Echo Reply 报文：

  1. 源主机发送 ICMP Echo Request 报文
  2. 目标主机收到后返回 ICMP Echo Reply 报文
  3. 源主机计算往返时间，判断网络连通性

Traceroute

Traceroute 程序使用 ICMP 超时报文：

  1. 发送 TTL 为 1 的 UDP 数据报
  2. 第一个路由器返回 ICMP 超时报文
  3. 逐渐增加 TTL 值，直到到达目标主机
  4. 记录每一跳的路由信息

ICMP 错误报告

ICMP 错误报告的类型：

  • 目的不可达：网络不可达、主机不可达、协议不可达、端口不可达等
  • 超时：TTL 超时或分片重组超时
  • 参数问题：IP 首部参数错误
  • 源站抑制：路由器拥塞，要求源主机降低发送速率

ICMP 重定向

当路由器发现更好的路由路径时，发送重定向报文：

  • 网络重定向：针对整个网络的重定向
  • 主机重定向：针对特定主机的重定向
  • 服务类型和网络重定向：基于服务类型的网络重定向
  • 服务类型和主机重定向：基于服务类型的主机重定向

ICMP 安全问题

ICMP 面临的安全威胁：

  • ICMP 洪水攻击：发送大量 ICMP 报文消耗目标资源
  • Ping of Death：发送超大 ICMP 报文导致系统崩溃
  • Smurf 攻击：利用 ICMP 广播放大攻击效果
  • ICMP 重定向攻击：伪造重定向报文改变路由路径

防御措施

防御 ICMP 攻击的措施：

  • 限制 ICMP 报文速率：防止洪水攻击
  • 过滤 ICMP 报文：阻止不必要的 ICMP 报文
  • 禁用 ICMP 重定向：防止重定向攻击
  • 使用防火墙：控制 ICMP 报文的进出

常见应用场景

ICMP 的常见应用场景：

  • 网络连通性测试：使用 ping 命令测试网络连接
  • 路由跟踪：使用 traceroute 命令查看路由路径
  • 网络故障诊断：分析 ICMP 错误信息定位网络问题
  • 网络性能测试：测量网络延迟和丢包率

常用网络命令

常用的网络诊断命令：

  • ping：测试网络连通性
  • traceroute/tracert：跟踪路由路径
  • pathping：结合 ping 和 traceroute 的功能
  • mtr：实时显示网络路径和性能

与 IP 协议的关系

ICMP 是 IP 协议的伴随协议：

  • ICMP 报文封装在 IP 数据报中
  • ICMP 使用 IP 进行传输
  • ICMP 不保证可靠传输
