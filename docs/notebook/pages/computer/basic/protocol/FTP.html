FTP 协议

FTP（File Transfer Protocol，文件传输协议）是用于在网络上进行文件传输的一套标准协议，使用客户/服务器模式。

工作原理

FTP 使用两个并行的 TCP 连接来传输文件：

  • 控制连接：用于传输控制信息（命令和响应），使用 21 端口
  • 数据连接：用于传输文件数据，使用 20 端口

工作模式

主动模式（PORT）

主动模式的工作流程：

  1. 客户端从随机端口 N 连接到服务器的 21 端口
  2. 客户端监听端口 N+1，并发送 PORT N+1 命令到服务器
  3. 服务器从 20 端口连接到客户端的 N+1 端口

被动模式（PASV）

被动模式的工作流程：

  1. 客户端从随机端口 N 连接到服务器的 21 端口
  2. 客户端发送 PASV 命令到服务器
  3. 服务器开启一个随机端口 P，并告知客户端
  4. 客户端从端口 N+1 连接到服务器的端口 P

常用 FTP 命令

常用的 FTP 命令包括：

  • USER：用户名
  • PASS：密码
  • LIST：列出目录内容
  • CWD：改变工作目录
  • PWD：显示当前工作目录
  • RETR：下载文件
  • STOR：上传文件
  • DELE：删除文件
  • MKD：创建目录
  • RMD：删除目录

FTP 响应码

FTP 响应码的含义：

  • 1xx：初步响应
  • 2xx：成功完成
  • 3xx：需要进一步信息
  • 4xx：暂时失败
  • 5xx：永久失败

常见应用场景

FTP 常见的应用场景：

  • 网站文件上传：将网站文件上传到服务器
  • 软件分发：分发大型软件安装包
  • 数据备份：定期备份重要数据
  • 文件共享：在组织内部共享文件

安全性问题

传统 FTP 协议存在以下安全问题：

  • 明文传输：用户名、密码和数据都以明文传输
  • 缺乏完整性保护：无法防止数据被篡改
  • 缺乏身份验证：无法验证服务器身份

安全替代方案

为了解决 FTP 的安全问题，可以使用以下安全替代方案：

  • FTPS：FTP over SSL/TLS，为 FTP 添加 SSL/TLS 加密
  • SFTP：SSH File Transfer Protocol，基于 SSH 的安全文件传输协议
  • SCP：Secure Copy，基于 SSH 的安全复制协议
