DNS 协议

DNS（Domain Name System，域名系统）是互联网的一项服务，它作为将域名和IP地址相互映射的一个分布式数据库，能够使人更方便地访问互联网。

DNS 使用客户/服务器模式，运行在 UDP 协议之上，使用 53 端口。当用户在浏览器中输入域名时，系统会向 DNS 服务器发送查询请求，获取对应的 IP 地址。

域名结构

域名采用层次结构，从右到左级别逐渐降低：

  • 根域：最顶层的域，用"."表示
  • 顶级域（TLD）：如 .com、.org、.net、.cn 等
  • 二级域：如 google.com 中的 google
  • 子域：如 mail.google.com 中的 mail

DNS 查询过程

DNS 查询的完整流程：

  1. 浏览器检查本地缓存是否有该域名对应的 IP
  2. 检查操作系统缓存（hosts 文件）
  3. 向本地 DNS 服务器发送查询请求
  4. 本地 DNS 服务器检查缓存，如果没有则进行递归查询
  5. 向根域名服务器查询
  6. 根域名服务器返回顶级域名服务器地址
  7. 向顶级域名服务器查询
  8. 顶级域名服务器返回权威域名服务器地址
  9. 向权威域名服务器查询
  10. 权威域名服务器返回最终的 IP 地址

DNS 记录类型

常见的 DNS 记录类型：

  • A 记录：将域名映射到 IPv4 地址
  • AAAA 记录：将域名映射到 IPv6 地址
  • CNAME 记录：别名记录，将一个域名映射到另一个域名
  • MX 记录：邮件交换记录，指定邮件服务器
  • NS 记录：域名服务器记录，指定该域名由哪个 DNS 服务器解析
  • TXT 记录：文本记录，可以存储任意文本信息
  • PTR 记录：指针记录，用于反向 DNS 查询

DNS 查询类型

DNS 查询的两种类型：

  • 递归查询：DNS 服务器为客户机完全解析域名（直到获得最终的 IP 地址）
  • 迭代查询：DNS 服务器为客户机提供下一步应该查询的 DNS 服务器地址

DNS 缓存

为了提高查询效率，DNS 系统采用多级缓存机制：

  • 浏览器缓存
  • 操作系统缓存
  • 本地 DNS 服务器缓存
  • ISP DNS 服务器缓存

DNS 安全问题

DNS 面临的主要安全威胁：

  • DNS 欺骗：攻击者伪造 DNS 响应，将用户引导到恶意网站
  • DNS 劫持：攻击者篡改 DNS 查询结果
  • DDoS 攻击：通过大量 DNS 查询请求使 DNS 服务器瘫痪

安全解决方案

DNS 安全解决方案：

  • DNSSEC：DNS 安全扩展，通过数字签名确保 DNS 响应的真实性和完整性
  • DoH：DNS over HTTPS，通过 HTTPS 加密 DNS 查询
  • DoT：DNS over TLS，通过 TLS 加密 DNS 查询

常见应用场景

DNS 的典型应用场景：

  • 网站访问：将域名解析为 IP 地址
  • 邮件传输：查找邮件服务器的 IP 地址
  • 负载均衡：通过 DNS 轮询实现简单的负载均衡
  • CDN 加速：根据用户地理位置返回最近的服务器 IP
