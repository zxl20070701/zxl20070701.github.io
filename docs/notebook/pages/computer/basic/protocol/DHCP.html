DHCP 协议

DHCP（Dynamic Host Configuration Protocol，动态主机配置协议）是一种网络管理协议，用于自动分配 IP 地址和其他网络配置参数给网络设备。

DHCP 使用客户/服务器模式，基于 UDP 协议，客户端使用 68 端口，服务器使用 67 端口。通过"租约"机制动态分配 IP 地址。

DHCP 工作流程

1. DHCP Discover（发现）

客户端广播 DHCP Discover 消息，寻找可用的 DHCP 服务器。

2. DHCP Offer（提供）

DHCP 服务器响应 DHCP Offer 消息，提供一个可用的 IP 地址和其他配置信息。

3. DHCP Request（请求）

客户端选择其中一个 Offer，并广播 DHCP Request 消息，请求使用该 IP 地址。

4. DHCP Acknowledge（确认）

被选中的 DHCP 服务器发送 DHCP Ack 消息，确认 IP 地址分配，并提供租约信息。

IP 地址租约

DHCP 分配的 IP 地址是有时间限制的，称为"租约"：

  • 租期：通常为几小时到几天
  • 续租：客户端在租期过半时尝试续租
  • 重绑定：如果续租失败，在租期 87.5% 时尝试重新获取
  • 释放：客户端可以主动释放 IP 地址

DHCP 消息类型

DHCP 定义了多种消息类型：

  • DHCPDISCOVER：客户端寻找 DHCP 服务器
  • DHCPOFFER：服务器提供配置参数
  • DHCPREQUEST：客户端请求配置参数
  • DHCPDECLINE：客户端拒绝配置参数
  • DHCPACK：服务器确认配置参数
  • DHCPNAK：服务器拒绝配置参数
  • DHCPRELEASE：客户端释放配置参数
  • DHCPINFORM：客户端请求本地配置参数

DHCP 选项（Options）

DHCP 消息包含各种配置选项：

  • 子网掩码（Option 1）
  • 路由器地址（Option 3）
  • DNS 服务器（Option 6）
  • 域名（Option 15）
  • IP 地址租期（Option 51）
  • DHCP 消息类型（Option 53）
  • 服务器标识符（Option 54）
  • 请求参数列表（Option 55）

DHCP 中继代理

当 DHCP 客户端和服务器不在同一个子网时，需要 DHCP 中继代理：

  • 中继代理接收客户端的广播消息
  • 将消息单播转发给 DHCP 服务器
  • 将服务器的响应转发回客户端

地址分配方式

DHCP 支持的地址分配方式：

  • 自动分配：DHCP 服务器为客户端分配永久 IP 地址
  • 动态分配：DHCP 服务器为客户端分配有租期的 IP 地址
  • 手动分配：管理员为特定客户端分配固定 IP 地址

优缺点

优点

  • 减少网络管理员工作量
  • 避免 IP 地址冲突
  • 灵活的网络配置管理
  • 支持移动设备

缺点

  • 单点故障风险
  • 可能存在安全风险
  • 需要专门的 DHCP 服务器

常见应用场景

DHCP 的常见应用场景：

  • 企业网络：为大量员工设备自动分配 IP
  • 公共 WiFi：为访客设备提供网络接入
  • 家庭网络：路由器为家庭设备分配 IP
  • 数据中心：为虚拟机动态分配网络配置
