VLAN

VLAN（Virtual Local Area Network，虚拟局域网）是一种将物理局域网划分为多个逻辑广播域的技术。通过VLAN，网络管理员可以根据功能、部门、应用等因素将网络设备逻辑分组，而不受物理位置的限制。VLAN技术提高了网络的安全性、可管理性和性能，是现代企业网络的基础技术之一。

VLAN基本概念

1. VLAN的定义

VLAN是在一个物理网络基础设施上创建的多个逻辑广播域，每个VLAN都是一个独立的广播域，VLAN间的通信需要通过三层设备（如路由器或三层交换机）进行。

2. VLAN的优势

  • 广播控制：限制广播域范围，减少广播流量
  • 安全性增强：不同VLAN间默认无法通信，提高安全性
  • 灵活管理：根据逻辑关系而非物理位置组织网络
  • 成本节约：减少路由器的使用，降低网络成本
  • 性能提升：减小广播域，提高网络整体性能

3. VLAN类型

3.1 基于端口的VLAN (Port-Based VLAN)

  • 原理：将交换机的物理端口分配给特定VLAN
  • 特点：配置简单，最常用
  • 应用：企业网络中最常见的VLAN类型

3.2 基于MAC地址的VLAN (MAC-Based VLAN)

  • 原理：根据设备的MAC地址分配VLAN
  • 特点：设备移动时VLAN成员身份保持不变
  • 应用：移动办公环境

3.3 基于协议的VLAN (Protocol-Based VLAN)

  • 原理：根据网络层协议类型分配VLAN
  • 特点：支持多协议网络
  • 应用：混合协议环境

3.4 基于子网的VLAN (Subnet-Based VLAN)

  • 原理：根据IP子网分配VLAN
  • 特点：与网络层紧密结合
  • 应用：大型网络环境

VLAN技术标准

1. IEEE 802.1Q标准

IEEE 802.1Q是VLAN技术的核心标准，定义了VLAN标签格式和操作方法。

1.1 VLAN标签格式 

| TPID (2字节) | TCI (2字节) |
| 0x8100       | PCP(3) | CFI(1) | VID(12) |
  • TPID (Tag Protocol Identifier)：标签协议标识符，固定为0x8100
  • TCI (Tag Control Information)：标签控制信息
    • PCP (Priority Code Point)：3位优先级，支持8个优先级(0-7)
    • CFI (Canonical Format Indicator)：1位规范格式指示符
    • VID (VLAN Identifier)：12位VLAN标识符，支持4096个VLAN(0-4095)

1.2 标记帧格式 

| 目的MAC(6) | 源MAC(6) | VLAN标签(4) | 类型/长度(2) | 数据 | FCS(4) |
  • 最大帧长度：1522字节(标准以太网1518字节 + 4字节VLAN标签)
  • 最小帧长度：68字节(64字节 + 4字节VLAN标签)

2. VLAN链路类型

2.1 访问链路 (Access Link)

  • 定义：连接终端设备的链路
  • 特点：只传输一个VLAN的流量，不带VLAN标签
  • 配置：将端口配置为access模式，指定VLAN ID

2.2 中继链路 (Trunk Link)

  • 定义：交换机之间或交换机与路由器之间的链路
  • 特点：可传输多个VLAN的流量，带VLAN标签
  • 标签协议：IEEE 802.1Q、Cisco ISL(私有)
  • 本征VLAN：Trunk链路上不带标签的VLAN(默认VLAN 1)

2.3 混合链路 (Hybrid Link)

  • 定义：既可连接终端设备又可连接其他交换机的链路
  • 特点：灵活配置，部分VLAN带标签，部分不带标签
  • 应用：复杂网络环境

VLAN配置与管理

1. VLAN创建与删除

1.1 创建VLAN

  • 全局配置模式：在全局配置模式下创建VLAN
  • VLAN数据库模式：在VLAN数据库模式下创建(传统方法)
  • 命名规则：VLAN ID范围1-4094，可配置名称

1.2 删除VLAN

  • 注意事项：删除VLAN前需将端口移出该VLAN
  • 默认VLAN：VLAN 1不能被删除

2. 端口配置

2.1 Access端口配置

  • 配置步骤：
    1. 进入接口配置模式
    2. 设置端口模式为access
    3. 指定VLAN成员身份
  • 验证命令：show vlan brief、show interfaces switchport

2.2 Trunk端口配置

  • 配置步骤：
    1. 进入接口配置模式
    2. 设置端口模式为trunk
    3. 配置允许的VLAN列表
    4. 设置本征VLAN(可选)
  • 封装协议：IEEE 802.1Q(标准)、ISL(Cisco私有)

3. VLAN间路由

3.1 传统路由器方案

  • 原理：每个VLAN连接到路由器的不同物理接口
  • 缺点：需要大量物理接口，成本高

3.2 单臂路由 (Router-on-a-Stick)

  • 原理：路由器通过单个物理接口连接交换机，使用子接口
  • 配置：在路由器上创建子接口，每个子接口对应一个VLAN
  • 封装：每个子接口配置802.1Q封装和VLAN ID
  • 优点：节省接口，成本较低

3.3 三层交换机方案

  • 原理：使用支持三层功能的交换机实现VLAN间路由
  • 配置：在交换机上创建SVI(Switched Virtual Interface)
  • 优点：性能高，配置简单，成本适中
  • 应用：现代企业网络主流方案

VLAN高级特性

1. Voice VLAN

  • 用途：为IP电话提供专用VLAN
  • 优势：服务质量(QoS)保证，安全隔离
  • 配置：将端口配置为同时支持数据和语音VLAN

2. Private VLAN

2.1 PVLAN类型

  • 主VLAN：Primary VLAN，PVLAN的父VLAN
  • 辅助VLAN：Secondary VLAN，包括隔离VLAN和社区VLAN

2.2 端口类型

  • 混杂端口：Promiscuous port，可与所有端口通信
  • 隔离端口：Isolated port，只能与混杂端口通信
  • 社区端口：Community port，可与同社区端口和混杂端口通信

2.3 应用场景

  • 服务器隔离：共享同一网段但需隔离的服务器
  • 托管服务：IDC中不同客户的隔离

3. QinQ (802.1ad)

  • 原理：在原有VLAN标签外再添加一层VLAN标签
  • 优势：扩展VLAN数量，支持服务提供商网络
  • 应用：运营商网络，企业专线服务

VLAN安全与管理

1. VLAN安全风险

1.1 VLAN跳跃攻击

  • 攻击方式：通过伪造VLAN标签或双重封装绕过VLAN隔离
  • 防护措施：禁用动态Trunk协议，配置本征VLAN

1.2 MAC地址泛洪攻击

  • 攻击方式：发送大量伪造MAC地址使交换机MAC表溢出
  • 防护措施：端口安全，限制MAC地址数量

1.3 广播风暴

  • 原因：二层环路或恶意攻击导致广播包无限循环
  • 防护措施：启用STP协议，配置广播风暴控制

2. VLAN管理最佳实践

2.1 VLAN规划原则

  • 按功能划分：根据部门、应用、安全等级划分VLAN
  • 地址规划：合理规划IP地址和VLAN的对应关系
  • 命名规范：使用有意义的VLAN名称

2.2 安全策略

  • 本征VLAN安全：将本征VLAN设置为未使用的VLAN
  • Trunk端口安全：限制允许的VLAN列表
  • VLAN ACL：配置VLAN访问控制列表

2.3 性能优化

  • VTP管理：使用VTP统一管理VLAN配置
  • STP优化：配置RSTP或MSTP提高收敛速度
  • 链路聚合：使用EtherChannel增加带宽

VLAN部署案例分析

1. 企业网络VLAN设计

典型的企业网络VLAN设计包括以下VLAN：

  • 管理VLAN：VLAN 1，用于网络设备管理
  • 用户VLAN：按部门划分，如VLAN 10(行政)、VLAN 20(财务)、VLAN 30(研发)
  • 服务器VLAN：VLAN 100，用于服务器接入
  • 语音VLAN：VLAN 200，用于IP电话
  • 访客VLAN：VLAN 300，用于访客网络，限制访问内部资源

2. 数据中心VLAN设计

  • 管理网络：独立的管理VLAN
  • 业务网络：按应用类型划分VLAN
  • 存储网络：独立的存储VLAN(iSCSI、FCoE)
  • 备份网络：独立的备份VLAN

3. 校园网VLAN设计

  • 按区域划分：教学楼、宿舍楼、图书馆等
  • 按用户类型：教师、学生、访客
  • 按应用类型：教学、科研、管理
