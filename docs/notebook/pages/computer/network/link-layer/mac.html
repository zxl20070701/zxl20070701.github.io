MAC

MAC（Media Access Control）地址是网络设备的物理地址，用于在局域网中唯一标识网络接口。MAC协议则负责控制网络设备如何访问共享传输介质，是数据链路层的重要组成部分。理解MAC地址和协议对于网络配置、故障排除和安全管理至关重要。

MAC地址结构

1. MAC地址格式

MAC地址是一个48位（6字节）的二进制数，通常以十六进制表示。

  • 标准格式：XX-XX-XX-XX-XX-XX或XX:XX:XX:XX:XX:XX
  • 示例：00-1A-2B-3C-4D-5E
  • 字节序：最高有效字节在前（大端序）

2. MAC地址组成

  • OUI (Organizationally Unique Identifier)：前24位，由IEEE分配给厂商
  • NIC Specific：后24位，由厂商分配给具体设备
  • 示例：00-1A-2B-3C-4D-5E中，00-1A-2B是OUI，3C-4D-5E是设备标识

3. MAC地址类型

3.1 单播地址 (Unicast Address)

  • 特征：第一个字节的最低位为0
  • 用途：标识单个网络接口
  • 示例：00-1A-2B-3C-4D-5E（00的二进制为00000000，最低位为0）

3.2 多播地址 (Multicast Address)

  • 特征：第一个字节的最低位为1
  • 用途：标识一组网络接口
  • 示例：01-00-5E-00-00-01（01的二进制为00000001，最低位为1）
  • IPv4多播映射：IPv4多播地址(224.0.0.0-239.255.255.255)映射到MAC地址01-00-5E-00-00-00到01-00-5E-7F-FF-FF

3.3 广播地址 (Broadcast Address)

  • 格式：FF-FF-FF-FF-FF-FF
  • 用途：发送到局域网内所有设备
  • 应用场景：ARP请求、DHCP发现等

3.4 特殊地址类型

  • 本地管理地址：第二个字节的次低位为1，用于本地管理
  • 全局唯一地址：第二个字节的次低位为0，由IEEE分配
  • 示例：02-00-00-00-00-01（本地管理单播地址）

MAC地址分配与管理

1. IEEE注册机构

  • IEEE RA：IEEE Registration Authority负责OUI分配
  • 申请流程：厂商向IEEE申请OUI，支付费用获得24位OUI
  • OUI数量：理论上可分配2²⁴=16,777,216个OUI

2. 地址分配策略

  • 顺序分配：按生产顺序分配连续的MAC地址
  • 随机分配：随机生成MAC地址，避免冲突
  • 虚拟化环境：虚拟机MAC地址通常使用本地管理地址

3. MAC地址表

  • 交换机MAC表：存储MAC地址与端口的映射关系
  • 老化机制：动态学习的MAC地址有老化时间(默认300秒)
  • 静态MAC：管理员手动配置的永久MAC地址映射

MAC协议原理

1. 介质访问控制分类

1.1 基于竞争的协议

  • 特点：多个站点竞争共享介质
  • 优点：简单灵活，适合突发流量
  • 缺点：存在冲突，效率随负载增加而下降
  • 代表协议：CSMA/CD、CSMA/CA

1.2 无竞争协议

  • 特点：通过调度算法避免冲突
  • 优点：确定性访问，无冲突
  • 缺点：复杂度高，需要中央控制
  • 代表协议：令牌传递、轮询

1.3 有限竞争协议

  • 特点：结合竞争和无竞争的优点
  • 工作机制：低负载时使用竞争协议，高负载时转为无竞争
  • 代表协议：自适应树协议

2. CSMA/CD协议详解

2.1 工作原理

  • 载波监听：发送前检测信道是否空闲
  • 冲突检测：发送过程中检测是否发生冲突
  • 冲突处理：检测到冲突后发送干扰信号并停止发送
  • 随机退避：等待随机时间后重试

2.2 二进制指数退避算法

  • 基本思想：冲突次数越多，等待时间越长
  • 计算公式：退避时间 = 随机数 × 时隙时间
  • 时隙时间：512位时间(10Mbps以太网为51.2μs)
  • 最大重传次数：通常为16次，超过后丢弃帧

2.3 性能分析

  • 吞吐量公式：S = G × e^(-2G)，其中G为网络负载
  • 最大吞吐量：当G=0.5时，S_max ≈ 0.184(18.4%)
  • 影响因素：网络负载、电缆长度、帧长度

3. CSMA/CA协议详解

3.1 无线环境特点

  • 隐藏节点问题：两个发送端无法听到对方，但接收端能同时收到
  • 暴露节点问题：发送端能听到其他通信，但接收端不会受到影响
  • 信号衰减：无线信号强度随距离增加而减弱

3.2 CSMA/CA机制

  • 帧间隔：DIFS(分布式协调IFS)、SIFS(短IFS)、PIFS(点协调IFS)
  • 虚拟载波监听：通过RTS/CTS帧预约信道
  • 网络分配向量：NAV指示信道将被占用的时间
  • 退避算法：二进制指数退避，但时隙时间较短

3.3 RTS/CTS握手过程

  1. 发送端：发送RTS(Request to Send)帧，包含预计传输时间
  2. 接收端：回复CTS(Clear to Send)帧，确认信道预约
  3. 其他站点：设置NAV，在指定时间内不发送数据
  4. 数据传输：发送端收到CTS后开始数据传输

以太网MAC子层

1. 以太网帧格式

  • 前导码：7字节10101010模式，用于时钟同步
  • 帧开始定界符：1字节10101011，标识帧开始
  • MAC地址：6字节目的MAC和6字节源MAC
  • 类型/长度：2字节，标识上层协议或数据长度
  • 数据：46-1500字节有效载荷
  • FCS：4字节CRC校验码

2. MAC地址解析

2.1 ARP协议

  • 功能：将IP地址解析为MAC地址
  • 工作过程：
    1. 发送ARP请求广播(目的MAC为FF-FF-FF-FF-FF-FF)
    2. 目标主机回复ARP响应单播
    3. 发送方缓存ARP映射关系
  • ARP缓存：存储IP-MAC映射，有老化机制

2.2 代理ARP

  • 功能：路由器代替远端主机响应ARP请求
  • 应用场景：子网间通信、透明网关

3. 以太网交换机工作原理

3.1 地址学习

  • 学习过程：交换机检查帧的源MAC地址，记录与端口的映射
  • MAC地址表：存储MAC地址、端口号、VLAN ID和时间戳
  • 老化机制：动态学习的条目超时删除(默认300秒)

3.2 帧转发决策

  • 已知单播：根据MAC地址表转发到对应端口
  • 未知单播：泛洪到所有端口(除接收端口)
  • 广播/多播：泛洪到所有端口(除接收端口)

3.3 环路防止

  • STP协议：生成树协议防止二层环路
  • BPDU：桥协议数据单元，用于STP通信
  • 根桥选举：选择优先级最低的交换机作为根桥

无线局域网MAC

1. IEEE 802.11 MAC架构

  • 分布式协调功能：DCF，基于CSMA/CA
  • 点协调功能：PCF，可选的集中式协调
  • 混合协调功能：HCF，QoS支持

2. 帧类型

2.1 管理帧

  • 信标帧：AP定期发送，包含网络信息
  • 关联/解除关联：站点与AP建立/断开连接
  • 认证帧：身份认证过程

2.2 控制帧

  • RTS/CTS：信道预约
  • ACK：数据帧确认
  • PS-Poll：省电模式轮询

2.3 数据帧

  • 数据：承载上层数据
  • 空帧：用于功率管理

3. 省电机制

  • 主动模式：持续监听信道
  • 省电模式：定期唤醒监听信标帧
  • TIM：传输指示图，指示AP是否有缓存数据

MAC安全技术

1. MAC地址过滤

  • 白名单：只允许指定MAC地址的设备接入
  • 黑名单：禁止指定MAC地址的设备接入
  • 局限性：MAC地址可伪造，安全性有限

2. 端口安全

  • 静态MAC绑定：将特定MAC绑定到交换机端口
  • 动态MAC限制：限制端口可学习的MAC地址数量
  • 违规处理：关闭端口、限制VLAN等

3. 802.1X认证

  • 体系结构：认证者(交换机)、请求者(客户端)、认证服务器(RADIUS)
  • 认证过程：基于端口的网络访问控制
  • 优势：用户级认证，支持多种认证方法
