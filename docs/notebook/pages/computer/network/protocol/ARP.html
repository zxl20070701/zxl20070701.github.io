ARP 协议

ARP（Address Resolution Protocol，地址解析协议）是根据 IP 地址获取物理地址（MAC 地址）的一个 TCP/IP 协议。

ARP 协议工作在数据链路层和网络层之间，用于将网络层的 32 位 IP 地址解析为数据链路层的 48 位 MAC 地址。

ARP 工作流程

1. ARP 请求

当主机需要发送数据包到某个 IP 地址时：

  1. 检查本地 ARP 缓存是否有该 IP 对应的 MAC 地址
  2. 如果没有，构造 ARP 请求报文
  3. 广播 ARP 请求到本地网络

2. ARP 响应

目标主机收到 ARP 请求后：

  1. 检查请求中的目标 IP 地址是否为自己的 IP
  2. 如果是，构造 ARP 响应报文
  3. 单播 ARP 响应给请求主机

3. 更新 ARP 缓存

请求主机收到响应后：

  1. 将 IP-MAC 映射关系存入 ARP 缓存
  2. 使用获得的 MAC 地址发送数据包

ARP 报文格式

ARP 报文包含以下字段：

  • 硬件类型：如以太网（值为 1）
  • 协议类型：如 IPv4（值为 0x0800）
  • 硬件地址长度：MAC 地址长度（6 字节）
  • 协议地址长度：IP 地址长度（4 字节）
  • 操作码：1 表示请求，2 表示响应
  • 发送方 MAC 地址
  • 发送方 IP 地址
  • 目标 MAC 地址
  • 目标 IP 地址

ARP 缓存

ARP 缓存存储 IP 地址到 MAC 地址的映射关系：

  • 动态条目：通过 ARP 协议学习获得，有超时时间
  • 静态条目：手动配置，不会过期
  • 查看 ARP 缓存：使用 arp -a 命令

ARP 代理

当网络被路由器分隔时，路由器可以作为 ARP 代理：

  • 路由器响应不同子网主机的 ARP 请求
  • 代替远程主机回答 ARP 请求
  • 使主机能够通信而不需要知道路由器的存在

反向 ARP（RARP）

RARP 与 ARP 相反，是根据 MAC 地址获取 IP 地址：

  • 主要用于无盘工作站获取 IP 地址
  • 已被 DHCP 协议取代

ARP 欺骗（ARP Spoofing）

ARP 协议的安全问题：

  • 攻击原理：攻击者发送伪造的 ARP 响应，将自己的 MAC 地址与他人的 IP 地址关联
  • 攻击后果：数据包被重定向到攻击者机器
  • 攻击类型：中间人攻击、拒绝服务攻击

防御 ARP 欺骗

防御 ARP 欺骗的方法：

  • 静态 ARP 表：手动配置 IP-MAC 映射
  • ARP 检测工具：监控网络中的 ARP 活动
  • 端口安全：交换机限制每个端口的 MAC 地址
  • 动态 ARP 检测：交换机验证 ARP 报文的合法性

常见应用场景

ARP 的常见应用场景：

  • 局域网通信：主机间通过 MAC 地址通信
  • 网关发现：获取默认网关的 MAC 地址
  • 网络诊断：排查网络连接问题
  • 负载均衡：某些负载均衡技术使用 ARP

ARP 命令

常用的 ARP 命令：

  • arp -a：显示 ARP 缓存表
  • arp -d：删除 ARP 缓存条目
  • arp -s：添加静态 ARP 条目
