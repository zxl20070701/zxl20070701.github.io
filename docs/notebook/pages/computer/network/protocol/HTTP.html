HTTP 协议

HTTP（HyperText Transfer Protocol，超文本传输协议）是互联网上应用最为广泛的一种网络协议，是客户端和服务器端之间请求和响应的标准协议。

HTTP 是一个基于请求与响应模式的、无状态的、应用层的协议，常基于 TCP/IP 协议传输数据。

HTTP 请求方法

HTTP 定义了多种请求方法，用于不同的操作：

  • GET：请求指定的页面信息，并返回实体主体
  • POST：向指定资源提交数据进行处理请求（例如提交表单或者上传文件）
  • PUT：从客户端向服务器传送的数据取代指定的文档的内容
  • DELETE：请求服务器删除指定的页面
  • HEAD：类似于 GET 请求，只不过返回的响应中没有具体的内容，用于获取报头
  • OPTIONS：允许客户端查看服务器的性能

HTTP 状态码

HTTP 状态码由三个十进制数字组成，第一个数字定义了状态码的类型：

  • 1xx：信息性状态码（如 100 Continue）
  • 2xx：成功状态码（如 200 OK）
  • 3xx：重定向状态码（如 301 Moved Permanently）
  • 4xx：客户端错误状态码（如 404 Not Found）
  • 5xx：服务器错误状态码（如 500 Internal Server Error）

HTTP 报文结构

HTTP 报文分为请求报文和响应报文两种，都包含以下三个部分：

  1. 起始行（start line）：描述请求或响应的基本信息
  2. 首部字段（header）：包含各种属性
  3. 主体（body）：包含实际传输的数据

常见应用场景

HTTP 协议广泛应用于以下场景：

  • 网页浏览
  • API 接口调用
  • 文件下载
  • 表单提交
