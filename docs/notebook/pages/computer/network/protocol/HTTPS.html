HTTPS 协议

HTTPS（HyperText Transfer Protocol Secure，安全超文本传输协议）是 HTTP 协议的安全版本，通过在 HTTP 和 TCP 之间加入 SSL/TLS 安全层来实现数据加密传输。

HTTPS 在 HTTP 的基础上加入了 SSL/TLS 协议，SSL/TLS 依靠证书来验证服务器的身份，并为浏览器和服务器之间的通信加密。

SSL/TLS 握手过程

SSL/TLS 握手是建立安全连接的关键步骤，过程如下：

  1. 客户端向服务器发送Client Hello消息
  2. 服务器回应Server Hello消息，并发送数字证书
  3. 客户端验证证书有效性
  4. 客户端生成随机密钥，用服务器公钥加密后发送给服务器
  5. 服务器用私钥解密获得密钥
  6. 双方使用此密钥进行对称加密通信

HTTPS 的优势

HTTPS 相比 HTTP 提供了以下安全优势：

  • 数据加密：防止数据在传输过程中被窃听
  • 身份验证：确保通信双方的身份真实性
  • 数据完整性：防止数据在传输过程中被篡改

证书类型

SSL/TLS 证书根据验证级别分为三种类型：

  • DV 证书（域名验证）：验证域名所有权
  • OV 证书（组织验证）：验证组织身份
  • EV 证书（扩展验证）：最严格的验证方式

HTTPS 端口号

HTTPS 默认使用 443 端口，区别于 HTTP 的 80 端口。

常见应用场景

HTTPS 广泛应用于需要保护数据安全的场景：

  • 网上银行
  • 电子商务网站
  • 用户登录页面
  • 涉及敏感信息传输的所有网站
