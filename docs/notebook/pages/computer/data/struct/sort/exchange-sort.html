交换排序

交换排序的定义

交换排序（Exchange Sort）是一类通过交换逆序元素来实现排序的算法。这类算法的基本思想是：两两比较待排序记录的关键字，发现两个记录的次序相反时即进行交换，直到没有逆序的记录为止。冒泡排序和快速排序是交换排序的两种典型代表。

交换排序的特点

  • 通过交换实现排序：通过交换逆序元素来达到排序目的
  • 就地排序：大部分交换排序算法只需要常数级别的额外空间
  • 实现相对简单：算法逻辑相对直观易懂

冒泡排序

冒泡排序的定义

冒泡排序（Bubble Sort）是一种简单的交换排序算法。它重复地走访过要排序的数列，一次比较两个元素，如果它们的顺序错误就把它们交换过来。走访数列的工作是重复地进行直到没有再需要交换，也就是说该数列已经排序完成。

冒泡排序的实现 

void BubbleSort(int arr[], int n) {
    int i, j, temp;
    bool swapped;

    for (i = 0; i < n - 1; i++) {
        swapped = false;

        // 最后i个元素已经到位
        for (j = 0; j < n - i - 1; j++) {
            if (arr[j] > arr[j + 1]) {
                // 交换相邻元素
                temp = arr[j];
                arr[j] = arr[j + 1];
                arr[j + 1] = temp;
                swapped = true;
            }
        }

        // 如果没有发生交换，说明数组已经有序
        if (!swapped) {
            break;
        }
    }
}

冒泡排序的优化

鸡尾酒排序（双向冒泡排序） 

void CocktailSort(int arr[], int n) {
    int i, temp;
    bool swapped = true;
    int start = 0;
    int end = n - 1;

    while (swapped) {
        swapped = false;

        // 从左到右的冒泡
        for (i = start; i < end; i++) {
            if (arr[i] > arr[i + 1]) {
                temp = arr[i];
                arr[i] = arr[i + 1];
                arr[i + 1] = temp;
                swapped = true;
            }
        }

        // 如果没有发生交换，说明数组已经有序
        if (!swapped) {
            break;
        }

        // 减少end，因为最大的元素已经到位
        end--;

        // 从右到左的冒泡
        for (i = end; i > start; i--) {
            if (arr[i] < arr[i - 1]) {
                temp = arr[i];
                arr[i] = arr[i - 1];
                arr[i - 1] = temp;
                swapped = true;
            }
        }

        // 增加start，因为最小的元素已经到位
        start++;
    }
}

快速排序

快速排序的定义

快速排序（Quick Sort）是对冒泡排序的一种改进。它的基本思想是：通过一趟排序将要排序的数据分割成独立的两部分，其中一部分的所有数据都比另外一部分的所有数据都要小，然后再按此方法对这两部分数据分别进行快速排序，整个排序过程可以递归进行，以此达到整个数据变成有序序列。

快速排序的实现 

void QuickSort(int arr[], int low, int high) {
    if (low < high) {
        // 找到分区点
        int pivot = Partition(arr, low, high);

        // 递归排序左半部分
        QuickSort(arr, low, pivot - 1);

        // 递归排序右半部分
        QuickSort(arr, pivot + 1, high);
    }
}

int Partition(int arr[], int low, int high) {
    int pivot = arr[high];  // 选择最后一个元素作为基准
    int i = low - 1;  // 较小元素的索引
    int temp;

    for (int j = low; j < high; j++) {
        // 如果当前元素小于或等于基准
        if (arr[j] <= pivot) {
            i++;
            temp = arr[i];
            arr[i] = arr[j];
            arr[j] = temp;
        }
    }

    // 将基准元素放到正确位置
    temp = arr[i + 1];
    arr[i + 1] = arr[high];
    arr[high] = temp;

    return i + 1;
}

快速排序的优化

三数取中法选择基准 

int MedianOfThree(int arr[], int low, int high) {
    int mid = low + (high - low) / 2;

    if (arr[mid] < arr[low]) {
        int temp = arr[low];
        arr[low] = arr[mid];
        arr[mid] = temp;
    }

    if (arr[high] < arr[low]) {
        int temp = arr[low];
        arr[low] = arr[high];
        arr[high] = temp;
    }

    if (arr[high] < arr[mid]) {
        int temp = arr[mid];
        arr[mid] = arr[high];
        arr[high] = temp;
    }

    // 将中位数放到倒数第二个位置
    int temp = arr[mid];
    arr[mid] = arr[high - 1];
    arr[high - 1] = temp;

    return arr[high - 1];
}

尾递归优化 

void QuickSortOptimized(int arr[], int low, int high) {
    while (low < high) {
        // 找到分区点
        int pivot = Partition(arr, low, high);

        // 对较小的部分进行递归排序
        if (pivot - low < high - pivot) {
            QuickSortOptimized(arr, low, pivot - 1);
            low = pivot + 1;
        } else {
            QuickSortOptimized(arr, pivot + 1, high);
            high = pivot - 1;
        }
    }
}

交换排序的性能分析

冒泡排序的性能分析

时间复杂度

  • 最好情况：O(n)，数据已经有序
  • 最坏情况：O(n²)，数据逆序排列
  • 平均情况：O(n²)

空间复杂度

O(1)，只需要常数级别的额外空间。

快速排序的性能分析

时间复杂度

  • 最好情况：O(n log n)，每次分区都很均匀
  • 最坏情况：O(n²)，每次分区都极不均匀
  • 平均情况：O(n log n)

空间复杂度

  • 最好情况：O(log n)
  • 最坏情况：O(n)

交换排序的应用场景

冒泡排序的应用场景

  • 小规模数据排序
  • 基本有序数据的排序
  • 教学演示，帮助理解排序算法的基本概念

快速排序的应用场景

  • 大规模数据排序
  • 对性能要求较高的场景
  • 需要原地排序的场景

交换排序的优缺点

冒泡排序的优缺点

优点

  • 实现简单，易于理解和维护
  • 稳定排序，保持相等元素的相对顺序
  • 原地排序，只需要常数级别的额外空间
  • 对于基本有序的数据，效率很高

缺点

  • 时间复杂度较高，为O(n²)
  • 对于大规模数据，效率较低
  • 元素交换次数较多

快速排序的优缺点

优点

  • 平均时间复杂度为O(n log n)，性能优秀
  • 原地排序，只需要常数级别的额外空间
  • 在实际应用中表现良好

缺点

  • 最坏情况下时间复杂度为O(n²)
  • 不稳定排序，可能改变相等元素的相对位置
  • 递归实现可能导致栈溢出
