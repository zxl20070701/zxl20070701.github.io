分配排序

分配排序的定义

分配排序（Distribution Sort）是一类不基于比较的排序算法，它们通过将元素分配到不同的桶中，然后按顺序收集这些桶中的元素来实现排序。这类算法通常利用元素的某些特性（如数值范围、位数等）来进行分配，因此可以达到线性时间复杂度O(n)。常见的分配排序算法包括计数排序、桶排序和基数排序。

分配排序的特点

  • 非比较排序：不通过元素间的比较来确定顺序
  • 线性时间复杂度：在特定条件下可以达到O(n)的时间复杂度
  • 稳定性：大多数分配排序算法是稳定的
  • 空间换时间：通常需要额外的存储空间

计数排序

计数排序（Counting Sort）是一种非比较排序算法，适用于一定范围内的整数排序。它的基本思想是：统计每个元素出现的次数，然后根据统计结果重构有序序列。

桶排序

桶排序（Bucket Sort）是将数组分到有限数量的桶里，每个桶再个别排序（有可能再使用别的排序算法或是以递归方式继续使用桶排序进行排序）。

基数排序

基数排序（Radix Sort）是按照低位先排序，然后收集；再按照高位排序，然后再收集；依次类推，直到最高位。有时候有些属性是有优先级顺序的，先按低优先级排序，再按高优先级排序。

分配排序的性能分析

计数排序的性能分析

时间复杂度

  • 最好情况：O(n + k)，其中k是数据范围
  • 最坏情况：O(n + k)
  • 平均情况：O(n + k)

空间复杂度

O(k)，需要额外的计数数组。

桶排序的性能分析

时间复杂度

  • 最好情况：O(n + k)，数据均匀分布
  • 最坏情况：O(n²)，所有数据在一个桶中
  • 平均情况：O(n + k)

空间复杂度

O(n + k)，需要额外的桶存储空间。

基数排序的性能分析

时间复杂度

  • 最好情况：O(d * (n + k))，其中d是位数，k是基数
  • 最坏情况：O(d * (n + k))
  • 平均情况：O(d * (n + k))

空间复杂度

O(n + k)，需要额外的存储空间。
