栈和队列的定义

栈的定义

栈（Stack）是只允许在一端进行插入或删除操作的线性表。首先，栈是一种线性表，但限定这种线性表只能在某一端进行插入和删除操作。栈顶（Top）：线性表允许进行插入删除的那一端。栈底（Bottom）：固定的，不允许进行插入和删除的另一端。空栈：不含任何元素的空表。

栈的特点

  • 后进先出（LIFO，Last In First Out）：最后被插入的元素最先被删除
  • 操作受限：只能在栈顶进行插入和删除操作
  • 递归实现：栈是实现递归的基础

栈的基本操作

  • InitStack(&S)：初始化一个空栈S
  • StackEmpty(S)：判断一个栈是否为空，若栈S为空返回true，否则返回false
  • Push(&S, x)：进栈，若栈S未满，将x加入使之成为新栈顶
  • Pop(&S, &x)：出栈，若栈S非空，弹出栈顶元素，并用x返回
  • GetTop(S, &x)：读栈顶元素，若栈S非空，用x返回栈顶元素
  • DestroyStack(&S)：销毁栈，并释放栈S占用的存储空间

队列的定义

队列（Queue）简称队，也是一种操作受限的线性表，只允许在表的一端进行插入，而在表的另一端进行删除。向队列中插入元素称为入队或进队；删除元素称为出队或离队。

队列的特点

  • 先进先出（FIFO，First In First Out）：最先插入的元素最先被删除
  • 队头出队：允许删除的一端称为队头（Front）
  • 队尾入队：允许插入的一端称为队尾（Rear）

队列的基本操作

  • InitQueue(&Q)：初始化队列，构造一个空队列Q
  • QueueEmpty(Q)：判队列空，若队列Q为空返回true，否则返回false
  • EnQueue(&Q, x)：入队，若队列Q未满，将x加入，使之成为新的队尾
  • DeQueue(&Q, &x)：出队，若队列Q非空，删除队头元素，并用x返回
  • GetHead(Q, &x)：读队头元素，若队列Q非空，则将队头元素赋值给x

栈和队列的比较

特性 栈 队列
操作原则 LIFO（后进先出） FIFO（先进先出）
插入位置 栈顶 队尾
删除位置 栈顶 队头
主要应用 函数调用、表达式求值、括号匹配 任务调度、缓冲区管理、广度优先搜索

栈和队列的抽象数据类型

栈的抽象数据类型 

ADT Stack {
    数据对象：D = {aᵢ | aᵢ ∈ ElemSet, i = 1, 2, ..., n, n ≥ 0}
    数据关系：R = {<aᵢ₋₁, aᵢ> | aᵢ₋₁, aᵢ ∈ D, i = 2, ..., n}
    约定aₙ端为栈顶，a₁端为栈底

    基本操作：
        InitStack(&S)
        DestroyStack(&S)
        StackEmpty(S)
        StackLength(S)
        GetTop(S, &e)
        ClearStack(&S)
        Push(&S, e)
        Pop(&S, &e)
}

队列的抽象数据类型 

ADT Queue {
    数据对象：D = {aᵢ | aᵢ ∈ ElemSet, i = 1, 2, ..., n, n ≥ 0}
    数据关系：R = {<aᵢ₋₁, aᵢ> | aᵢ₋₁, aᵢ ∈ D, i = 2, ..., n}
    约定其中a₁端为队列头，aₙ端为队列尾

    基本操作：
        InitQueue(&Q)
        DestroyQueue(&Q)
        QueueEmpty(Q)
        QueueLength(Q)
        GetHead(Q, &e)
        ClearQueue(&Q)
        EnQueue(&Q, e)
        DeQueue(&Q, &e)
}

栈和队列的数学性质

栈的数学性质

栈的操作序列可以用数学方法进行分析。对于一个栈，如果已知入栈序列和出栈序列，可以判断该出栈序列是否合法。合法的出栈序列必须满足：对于任意元素，在它出栈之前，所有在它之后入栈且尚未出栈的元素必须按照逆序出栈。

队列的数学性质

队列的操作序列相对简单，因为队列是严格的先进先出结构。给定入队序列，出队序列是唯一确定的，且与入队序列相同。
