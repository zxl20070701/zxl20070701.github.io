最小生成树

最小生成树的定义

最小生成树（Minimum Spanning Tree，MST）是在一个连通无向图中，包含图中所有顶点的一棵生成树，且所有边的权值之和最小。生成树是图的极小连通子图，它包含图中全部顶点，但只有足以构成一棵树的n-1条边。

相关概念

  • 生成树：包含图中所有顶点的极小连通子图
  • 最小生成树：所有生成树中边的权值之和最小的生成树
  • 连通图：任意两个顶点之间都有路径的图

最小生成树的性质

性质1（切割性质）

对于图的任意一个切割（将图的顶点分为两个不相交的集合），连接两个集合的最小权值边一定属于某个最小生成树。

性质2（环性质）

对于图中的任意一个环，环上权值最大的边一定不属于任何最小生成树。

最小生成树的算法

Prim算法

算法思想

Prim算法是一种贪心算法，其基本思想是：从图中任意一个顶点开始，每次选择一条连接已选顶点集合和未选顶点集合的最小权值边，将这条边和对应的顶点加入到已选集合中，直到所有顶点都被选中。

算法步骤

  1. 初始化：选择一个起始顶点，将其加入已选集合，其余顶点加入未选集合
  2. 重复以下步骤直到所有顶点都被选中：
    1. 在连接已选顶点和未选顶点的所有边中，选择权值最小的边
    2. 将这条边和对应的未选顶点加入到已选集合中

Prim算法的实现 

#define MAX_VERTEX_NUM 100
#define INFINITY 65535

void MiniSpanTree_Prim(MGraph G) {
    int min, i, j, k;
    int adjvex[MAX_VERTEX_NUM];  // 保存相关顶点下标
    int lowcost[MAX_VERTEX_NUM]; // 保存相关顶点间边的权值

    lowcost[0] = 0;  // 初始化第一个权值为0，即v0加入生成树
    adjvex[0] = 0;   // 初始化第一个顶点下标为0

    // 循环除下标为0外的全部顶点
    for (i = 1; i < G.numVertexes; i++) {
        lowcost[i] = G.arc[0][i];  // 将v0顶点与之有边的权值存入数组
        adjvex[i] = 0;             // 初始化都为v0的下标
    }

    for (i = 1; i < G.numVertexes; i++) {
        min = INFINITY;
        j = 1;
        k = 0;

        // 遍历全部顶点
        while (j < G.numVertexes) {
            if (lowcost[j] != 0 && lowcost[j] < min) {
                min = lowcost[j];
                k = j;  // 将当前最小权值的下标存入k
            }
            j++;
        }

        printf("(%d, %d)\n", adjvex[k], k);  // 打印当前顶点边中权值最小的边
        lowcost[k] = 0;  // 将当前顶点的权值设为0，表示此顶点已经完成任务

        // 循环所有顶点
        for (j = 1; j < G.numVertexes; j++) {
            if (lowcost[j] != 0 && G.arc[k][j] < lowcost[j]) {
                lowcost[j] = G.arc[k][j];
                adjvex[j] = k;
            }
        }
    }
}

Kruskal算法

算法思想

Kruskal算法也是一种贪心算法，其基本思想是：将图中的所有边按权值从小到大排序，然后依次选择权值最小的边，如果这条边不会与已选择的边构成环，则将其加入到最小生成树中，直到选择了n-1条边为止。

算法步骤

  1. 将图中的所有边按权值从小到大排序
  2. 初始化一个空的最小生成树
  3. 依次考察每条边，如果这条边不会与已选择的边构成环，则将其加入到最小生成树中
  4. 重复步骤3，直到选择了n-1条边

Kruskal算法的实现 

typedef struct {
    int begin;
    int end;
    int weight;
} Edge;

void MiniSpanTree_Kruskal(MGraph G) {
    int i, n, m;
    Edge edges[MAX_EDGE];  // 定义边集数组
    int parent[MAX_VERTEX_NUM];  // 定义一数组用来判断边与边是否形成环路

    // 此处省略将邻接矩阵G转换为边集数组edges并按权值从小到大排序的代码

    for (i = 0; i < G.numVertexes; i++) {
        parent[i] = 0;  // 初始化数组值为0
    }

    for (i = 0; i < G.numEdges; i++) {
        n = Find(parent, edges[i].begin);
        m = Find(parent, edges[i].end);

        if (n != m) {  // 如果n与m不等，说明此边没有与现有生成树形成环路
            parent[n] = m;  // 将此边的结尾顶点放入下标为起点的parent中
            printf("(%d, %d) %d\n", edges[i].begin, edges[i].end, edges[i].weight);
        }
    }
}

int Find(int *parent, int f) {
    while (parent[f] > 0) {
        f = parent[f];
    }
    return f;
}

最小生成树的应用

网络设计

在计算机网络、通信网络、电力网络等设计中，最小生成树可以用于设计成本最低的网络连接方案。

聚类分析

在数据挖掘中，最小生成树可以用于聚类分析，识别数据中的自然分组。

图像分割

在计算机视觉中，最小生成树可以用于图像分割，将图像分割成不同的区域。

路径规划

在机器人学和地理信息系统中，最小生成树可以用于路径规划，找到连接多个目标点的最优路径。

最小生成树的性质

唯一性

如果图中所有边的权值都不相同，那么最小生成树是唯一的。

最优性

最小生成树是所有生成树中边的权值之和最小的。

割性质

对于图的任意一个割，连接两个部分的最小权值边一定属于某个最小生成树。

最小生成树的扩展

次小生成树

在所有生成树中，边的权值之和第二小的生成树。

最小瓶颈生成树

在所有生成树中，最大边权值最小的生成树。

度限制最小生成树

在最小生成树的基础上，对某些顶点的度数进行限制。
