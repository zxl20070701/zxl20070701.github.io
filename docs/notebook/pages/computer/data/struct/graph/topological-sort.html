拓扑排序

拓扑排序的定义

拓扑排序（Topological Sorting）是对有向无环图（Directed Acyclic Graph，DAG）的一种线性排序。它将图中的所有顶点排成一个线性序列，使得对于图中的任意一条有向边(u,v)，u在序列中都出现在v的前面。如果图中有环，则无法进行拓扑排序。

相关概念

  • 有向无环图（DAG）：没有环的有向图
  • 前驱：在有向图中，如果存在边(u,v)，则u是v的前驱
  • 后继：在有向图中，如果存在边(u,v)，则v是u的后继
  • 入度：顶点的入度是指向该顶点的边的数量
  • 出度：顶点的出度是从该顶点出发的边的数量

拓扑排序的性质

存在性

一个图可以进行拓扑排序的充要条件是它是一个有向无环图。

唯一性

拓扑排序的结果不一定是唯一的，一个图可能有多个不同的拓扑排序结果。

应用性

拓扑排序在任务调度、课程安排、编译依赖分析等领域有广泛应用。

拓扑排序的算法

Kahn算法（基于入度）

算法思想

Kahn算法的基本思想是：不断选择入度为0的顶点，将其加入结果序列，并删除该顶点及其所有出边，直到所有顶点都被处理或发现图中存在环。

算法步骤

  1. 计算图中所有顶点的入度
  2. 将所有入度为0的顶点加入队列
  3. 当队列不为空时：
    1. 从队列中取出一个顶点，将其加入结果序列
    2. 删除该顶点及其所有出边，更新相关顶点的入度
    3. 如果某个顶点的入度变为0，将其加入队列
  4. 如果结果序列中的顶点数少于图中的顶点数，则说明图中存在环

Kahn算法的实现 

#define MAX_VERTEX_NUM 100

Status TopologicalSort(GraphAdjList GL) {
    EdgeNode *e;
    int i, k, gettop;
    int top = 0;  // 用于栈指针下标
    int count = 0;  // 用于统计输出顶点的个数
    int *stack;  // 建栈存储入度为0的顶点
    stack = (int*)malloc(GL->numVertexes * sizeof(int));

    // 遍历邻接表，计算每个顶点的入度
    for (i = 0; i < GL->numVertexes; i++) {
        if (GL->adjList[i].in == 0) {
            stack[++top] = i;  // 将入度为0的顶点入栈
        }
    }

    while (top != 0) {
        gettop = stack[top--];  // 出栈
        printf("%d -> ", GL->adjList[gettop].data);
        count++;  // 统计输出顶点数

        // 遍历邻接表，将以该顶点为尾的弧的头顶点的入度减1
        for (e = GL->adjList[gettop].firstedge; e; e = e->next) {
            k = e->adjvex;
            if (!(--GL->adjList[k].in)) {
                stack[++top] = k;  // 若入度为0，则入栈
            }
        }
    }

    if (count < GL->numVertexes) {
        return ERROR;  // 如果count小于顶点数，说明存在环
    } else {
        return OK;
    }
}

DFS算法（基于深度优先搜索）

算法思想

DFS算法的基本思想是：对图进行深度优先搜索，在回溯时将顶点加入结果序列。这样可以保证对于任意边(u,v)，u在序列中出现在v的前面。

算法步骤

  1. 对图中的每个未访问的顶点进行深度优先搜索
  2. 在回溯时将顶点加入结果序列
  3. 最终得到的结果序列的逆序就是拓扑排序的结果

DFS算法的实现 

#define MAX_VERTEX_NUM 100
int visited[MAX_VERTEX_NUM];  // 访问标志数组
int result[MAX_VERTEX_NUM];   // 存储拓扑排序结果
int index = 0;                // 结果数组下标

Status TopologicalSort_DFS(GraphAdjList GL) {
    int i;

    // 初始化访问标志数组
    for (i = 0; i < GL->numVertexes; i++) {
        visited[i] = 0;
    }

    // 对未访问的顶点进行深度优先搜索
    for (i = 0; i < GL->numVertexes; i++) {
        if (!visited[i]) {
            if (!DFS(GL, i)) {
                return ERROR;  // 发现环
            }
        }
    }

    // 输出拓扑排序结果（逆序）
    for (i = index - 1; i >= 0; i--) {
        printf("%d -> ", GL->adjList[result[i]].data);
    }

    return OK;
}

Status DFS(GraphAdjList GL, int i) {
    EdgeNode *e;
    int k;

    visited[i] = 1;  // 标记为已访问

    // 遍历所有邻接顶点
    for (e = GL->adjList[i].firstedge; e; e = e->next) {
        k = e->adjvex;
        if (!visited[k]) {
            if (!DFS(GL, k)) {
                return ERROR;  // 发现环
            }
        } else if (visited[k] == 1) {
            return ERROR;  // 发现环
        }
    }

    visited[i] = 2;  // 标记为已完成
    result[index++] = i;  // 将顶点加入结果序列

    return OK;
}

两种算法的比较

特性 Kahn算法 DFS算法
算法思想 基于入度 基于深度优先搜索
时间复杂度 O(V+E) O(V+E)
空间复杂度 O(V) O(V)
实现复杂度 中等 较高
环检测 自然检测 需要额外判断

其中，V是顶点数，E是边数。

拓扑排序的应用

任务调度

在项目管理中，拓扑排序可以用于确定任务的执行顺序，确保前置任务在后续任务之前完成。

课程安排

在教育系统中，拓扑排序可以用于安排课程，确保先修课程在后续课程之前开设。

编译依赖分析

在编译系统中，拓扑排序可以用于分析源文件之间的依赖关系，确定编译顺序。

数据流分析

在程序分析中，拓扑排序可以用于数据流分析，确定变量的定义和使用顺序。

拓扑排序的扩展

字典序拓扑排序

在满足拓扑排序条件的前提下，使结果序列字典序最小。

并行拓扑排序

利用并行计算技术加速拓扑排序过程。

增量拓扑排序

在图结构发生变化时，动态维护拓扑排序结果。
