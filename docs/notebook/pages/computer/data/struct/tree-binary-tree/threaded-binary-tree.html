线索二叉树

线索二叉树的定义

在二叉树的链式存储结构中，n个结点的二叉链表中含有n+1个空指针域。利用二叉链表中的空指针域，存放指向结点在某种遍历次序下的前驱和后继结点的指针（这种附加的指针称为"线索"）。这种加上了线索的二叉链表称为线索链表，相应的二叉树称为线索二叉树（Threaded Binary Tree）。

线索化

对二叉树以某种次序遍历使其变为线索二叉树的过程称为线索化。根据线索性质的不同，线索二叉树可分为前序线索二叉树、中序线索二叉树和后序线索二叉树三种。

线索二叉树的存储结构

结点结构 

typedef enum {Link, Thread} PointerTag;  // Link==0：指针，Thread==1：线索

typedef struct BiThrNode {
    int data;                        // 数据域
    struct BiThrNode *lchild, *rchild; // 左右孩子指针
    PointerTag LTag, RTag;          // 左右标志
} BiThrNode, *BiThrTree;

标志域的含义

  • LTag = Link：lchild指向结点的左孩子
  • LTag = Thread：lchild指向结点的直接前驱
  • RTag = Link：rchild指向结点的右孩子
  • RTag = Thread：rchild指向结点的直接后继

中序线索二叉树

中序线索化

中序线索化是在中序遍历的过程中修改空指针，使其指向相应的前驱或后继。 

BiThrTree pre = NULL;  // 全局变量，始终指向刚刚访问过的结点

void InThreading(BiThrTree p) {
    if (p) {
        InThreading(p->lchild);  // 递归左子树线索化

        if (!p->lchild) {        // 没有左孩子
            p->LTag = Thread;
            p->lchild = pre;
        }
        if (pre && !pre->rchild) {  // 前驱没有右孩子
            pre->RTag = Thread;
            pre->rchild = p;
        }
        pre = p;                 // 保持pre指向p的前驱

        InThreading(p->rchild);  // 递归右子树线索化
    }
}

构建中序线索二叉树 

Status InOrderThreading(BiThrTree *Thrt, BiThrTree T) {
    *Thrt = (BiThrTree)malloc(sizeof(BiThrNode));
    if (!*Thrt) return ERROR;

    (*Thrt)->LTag = Link;  // 头结点
    (*Thrt)->RTag = Thread;
    (*Thrt)->rchild = *Thrt;  // 右指针回指

    if (!T) {
        (*Thrt)->lchild = *Thrt;
    } else {
        (*Thrt)->lchild = T;
        pre = *Thrt;
        InThreading(T);
        pre->rchild = *Thrt;     // 最后一个结点线索化
        pre->RTag = Thread;
        (*Thrt)->rchild = pre;
    }

    return OK;
}

中序遍历线索二叉树 

Status InOrderTraverse_Thr(BiThrTree T) {
    BiThrTree p = T->lchild;  // p指向根结点

    while (p != T) {
        while (p->LTag == Link) p = p->lchild;  // 找到第一个结点

        printf("%d ", p->data);  // 访问结点

        while (p->RTag == Thread && p->rchild != T) {
            p = p->rchild;       // 访问后继结点
            printf("%d ", p->data);
        }

        p = p->rchild;  // p进至其右子树根
    }

    return OK;
}

前序线索二叉树

前序线索化 

void PreThreading(BiThrTree p) {
    if (p) {
        if (!p->lchild) {
            p->LTag = Thread;
            p->lchild = pre;
        }
        if (pre && !pre->rchild) {
            pre->RTag = Thread;
            pre->rchild = p;
        }
        pre = p;

        if (p->LTag == Link) PreThreading(p->lchild);
        if (p->RTag == Link) PreThreading(p->rchild);
    }
}

后序线索二叉树

后序线索化 

void PostThreading(BiThrTree p) {
    if (p) {
        PostThreading(p->lchild);
        PostThreading(p->rchild);

        if (!p->lchild) {
            p->LTag = Thread;
            p->lchild = pre;
        }
        if (pre && !pre->rchild) {
            pre->RTag = Thread;
            pre->rchild = p;
        }
        pre = p;
    }
}

线索二叉树的优势

无需栈或递归

在线索二叉树中，可以直接通过线索找到前驱和后继结点，无需使用栈或递归进行遍历。

高效的遍历

线索二叉树的遍历效率高于普通二叉树，特别是对于频繁遍历的应用场景。

空间利用率高

利用了原本空闲的指针域，提高了空间利用率。

线索二叉树的操作

查找前驱结点 

BiThrTree InPreNode(BiThrTree p) {
    BiThrTree pre;
    if (p->LTag == Thread) {
        pre = p->lchild;
    } else {
        pre = p->lchild;
        while (pre->RTag == Link) {
            pre = pre->rchild;
        }
    }
    return pre;
}

查找后继结点 

BiThrTree InNextNode(BiThrTree p) {
    BiThrTree next;
    if (p->RTag == Thread) {
        next = p->rchild;
    } else {
        next = p->rchild;
        while (next->LTag == Link) {
            next = next->lchild;
        }
    }
    return next;
}

线索二叉树的应用场景

  • 数据库索引：B+树的叶子结点可以通过线索连接
  • 表达式求值：表达式树的中序线索化便于中缀表达式求值
  • 文件系统：目录结构的遍历
  • 编译器设计：语法树的处理

线索二叉树的优缺点

优点

  • 遍历无需栈或递归，节省空间
  • 遍历效率高
  • 可以方便地找到前驱和后继结点
  • 空间利用率高

缺点

  • 实现复杂度高
  • 插入和删除操作复杂
  • 线索化过程需要额外时间
  • 破坏了原有的树结构
