B树/B+树

B树的定义

B树（B-Tree）是一种平衡的多路搜索树，广泛应用于文件系统和数据库系统中。一棵m阶的B树，或为空树，或为满足下列特性的m叉树：

  1. 树中每个结点至多有m棵子树
  2. 若根结点不是叶子结点，则至少有两棵子树
  3. 除根之外的所有非终端结点至少有⌈m/2⌉棵子树
  4. 所有的非终端结点中包含下列信息数据：(n, A₀, K₁, A₁, K₂, A₂, ..., Kₙ, Aₙ)
  5. 所有的叶子结点都出现在同一层次上，并且不带信息（可以看作是外部结点或查找失败的结点，实际上这些结点不存在，指向这些结点的指针为空）

B树的结点结构 

#define m 3  // B树的阶，即最多子树个数
#define MAX m-1  // 最多关键字个数
#define MIN (m-1)/2  // 最少关键字个数

typedef struct BTNode {
    int keynum;                 // 结点中关键字的个数
    int key[m];                 // 关键字向量，key[0]未用
    struct BTNode *parent;      // 指向双亲结点
    struct BTNode *ptr[m+1];    // 子树指针向量
} BTNode, *BTree;

B树的性质

平衡性

B树是一种平衡树，所有叶子结点都在同一层上，保证了查找、插入和删除操作的时间复杂度都是对数级别的。

多路性

B树的每个结点可以有多个子树，这使得B树的高度相对较低，从而减少了磁盘I/O次数。

有序性

B树中的关键字是有序的，左子树中的所有关键字都小于根结点的关键字，右子树中的所有关键字都大于根结点的关键字。

B树的基本操作

查找操作

B树的查找过程是一个顺着指针查找结点和在结点内查找关键字交叉进行的过程。 

BTree SearchBTree(BTree T, int key, int *pos) {
    int i = 0;
    BTree p = T;

    while (p) {
        i = 0;
        p->key[0] = key;  // 哨兵

        while (p->key[i+1] < key) i++;

        if (p->key[i+1] == key && i + 1 <= p->keynum) {
            *pos = i + 1;
            return p;
        }

        p = p->ptr[i];
    }

    return NULL;
}

插入操作

B树的插入操作包括两个步骤：首先找到插入位置，然后插入关键字。如果插入后结点的关键字个数超过m-1，则需要进行分裂操作。 

Status InsertBTree(BTree *T, int key) {
    int finished = 0;
    int i;
    int s;
    BTree q;
    BTree p = *T;

    // 查找插入位置
    while (p && !finished) {
        i = 0;
        p->key[0] = key;

        while (p->key[i+1] < key) i++;

        if (p->key[i+1] == key && i + 1 <= p->keynum) {
            return DUPLICATE;  // 关键字已存在
        }

        if (p->ptr[i] == NULL) {
            // 叶子结点，可以插入
            if (p->keynum < MAX) {
                // 直接插入
                for (int j = p->keynum; j > i; j--) {
                    p->key[j+1] = p->key[j];
                    p->ptr[j+1] = p->ptr[j];
                }
                p->key[i+1] = key;
                p->ptr[i+1] = NULL;
                p->keynum++;
                finished = 1;
            } else {
                // 需要分裂
                InsertKey(p, key, NULL);
                finished = 1;
            }
        } else {
            p = p->ptr[i];
        }
    }

    return OK;
}

删除操作

B树的删除操作比插入操作更复杂，需要考虑多种情况。当删除关键字后，如果结点的关键字个数少于⌈m/2⌉-1，则需要进行合并或借关键字操作。

B+树

B+树的定义

B+树是B树的一种变体，也是一种平衡的多路搜索树。与B树相比，B+树有以下特点：

  • 有n棵子树的结点中含有n个关键字
  • 所有的叶子结点中包含了全部关键字的信息，及指向含这些关键字记录的指针，且叶子结点本身依关键字的大小自小而大顺序链接
  • 所有的非终端结点可以看成是索引部分，结点中仅含有其子树（根结点）中的最大（或最小）关键字

B+树与B树的区别

特性 B树 B+树
关键字个数 有n棵子树的结点有n-1个关键字 有n棵子树的结点有n个关键字
数据存储 关键字和数据都存储在所有结点中 关键字和数据只存储在叶子结点中
叶子结点 叶子结点不包含数据 叶子结点包含全部数据，并通过指针链接
查找 可能在非叶子结点结束 查找必须到叶子结点结束

B树和B+树的应用

数据库索引

B+树是数据库系统中最常用的索引结构，如MySQL的InnoDB存储引擎就使用B+树作为索引。

文件系统

许多文件系统使用B树或B+树来管理文件和目录，如NTFS、HFS+等。

操作系统

操作系统的虚拟内存管理、页表管理等也使用B树结构。

B树和B+树的性能分析

时间复杂度

  • 查找：O(log n)
  • 插入：O(log n)
  • 删除：O(log n)

空间复杂度

  • 存储空间：O(n)
  • 内存使用：相对较低，因为树的高度较小

B树和B+树的优缺点

B树的优点

  • 查找效率稳定
  • 插入和删除操作相对简单
  • 适合随机访问

B树的缺点

  • 非叶子结点存储数据，空间利用率相对较低
  • 范围查询效率不如B+树

B+树的优点

  • 叶子结点通过指针链接，范围查询效率高
  • 非叶子结点只存储索引，空间利用率高
  • 所有查询都要到叶子结点结束，查询性能稳定

B+树的缺点

  • 插入和删除操作比B树复杂
  • 非叶子结点不存储数据，需要一次额外的磁盘I/O
