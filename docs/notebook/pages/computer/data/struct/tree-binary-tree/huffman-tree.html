哈夫曼树

哈夫曼树的定义

哈夫曼树（Huffman Tree），又称最优二叉树，是一类带权路径长度最短的树。给定n个权值作为n个叶子结点，构造一棵二叉树，若该树的带权路径长度达到最小，则称这样的二叉树为最优二叉树，也称为哈夫曼树。

相关概念

  • 路径：从树中一个结点到另一个结点之间的分支构成这两个结点间的路径
  • 路径长度：路径上的分支数目称为路径长度
  • 树的路径长度：从树根到每一结点的路径长度之和
  • 结点的带权路径长度：从该结点到树根之间的路径长度与结点上权的乘积
  • 树的带权路径长度：树中所有叶子结点的带权路径长度之和

哈夫曼树的构造算法

哈夫曼算法

  1. 根据给定的n个权值{w₁, w₂, ..., wₙ}构成n棵二叉树的集合F={T₁, T₂, ..., Tₙ}，其中每棵二叉树Tᵢ中只有一个带权为wᵢ的根结点，其左右子树均为空
  2. 在F中选取两棵根结点的权值最小的树作为左右子树构造一棵新的二叉树，且置新的二叉树的根结点的权值为其左、右子树上根结点的权值之和
  3. 在F中删除这两棵树，同时将新得到的二叉树加入F中
  4. 重复步骤2和3，直到F只含有一棵树为止。这棵树便是哈夫曼树

哈夫曼树的构造实现 

void CreateHuffmanTree(HuffmanTree *HT, int *w, int n) {
    if (n <= 1) return;

    int m = 2 * n - 1;  // 哈夫曼树结点总数
    *HT = (HuffmanTree)malloc((m + 1) * sizeof(HTNode));  // 0号单元未用

    // 初始化
    for (int i = 1; i <= n; i++) {
        (*HT)[i].weight = w[i-1];
        (*HT)[i].parent = 0;
        (*HT)[i].lchild = 0;
        (*HT)[i].rchild = 0;
    }
    for (int i = n + 1; i <= m; i++) {
        (*HT)[i].weight = 0;
        (*HT)[i].parent = 0;
        (*HT)[i].lchild = 0;
        (*HT)[i].rchild = 0;
    }

    // 构造哈夫曼树
    for (int i = n + 1; i <= m; i++) {
        int s1, s2;
        Select(*HT, i - 1, &s1, &s2);  // 选择parent为0且weight最小的两个结点

        (*HT)[s1].parent = i;
        (*HT)[s2].parent = i;
        (*HT)[i].lchild = s1;
        (*HT)[i].rchild = s2;
        (*HT)[i].weight = (*HT)[s1].weight + (*HT)[s2].weight;
    }
}

哈夫曼编码

哈夫曼编码的定义

哈夫曼编码（Huffman Coding）是一种编码方式，也是一种可变字长编码（VLC）。哈夫曼编码使用变长编码表对源符号进行编码，其中变长编码表是通过一种评估源符号出现概率的方法得到的。出现概率高的字母使用较短的编码，反之出现概率低的则使用较长的编码，这便使编码之后的字符串的平均长度、期望值降低，从而达到无损压缩数据的目的。

哈夫曼编码的特点

  • 前缀编码：任何一个字符的编码都不是另一个字符编码的前缀
  • 最优编码：对于给定的字符集和概率分布，哈夫曼编码是最优的前缀编码
  • 变长编码：不同字符的编码长度不同

哈夫曼编码的生成 

void CreateHuffmanCode(HuffmanTree HT, HuffmanCode *HC, int n) {
    *HC = (HuffmanCode)malloc((n + 1) * sizeof(char*));  // 分配n个字符编码的头指针向量
    char *cd = (char*)malloc(n * sizeof(char));          // 分配求编码的工作空间
    cd[n - 1] = '\0';                                    // 编码结束符

    for (int i = 1; i <= n; i++) {  // 逐个求哈夫曼编码
        int start = n - 1;          // 编码结束符位置
        int c = i;
        int f = HT[i].parent;

        while (f != 0) {  // 从叶子到根逆向求编码
            --start;
            if (HT[f].lchild == c) {
                cd[start] = '0';
            } else {
                cd[start] = '1';
            }
            c = f;
            f = HT[f].parent;
        }

        (*HC)[i] = (char*)malloc((n - start) * sizeof(char));
        strcpy((*HC)[i], &cd[start]);
    }

    free(cd);
}

哈夫曼树的性质

最优性

哈夫曼树是所有可能的二叉树中带权路径长度最小的二叉树。

唯一性

当所有字符的出现频率都不相同时，哈夫曼编码是唯一的。如果有多个字符具有相同的频率，则哈夫曼编码可能不唯一。

前缀性质

哈夫曼编码是前缀编码，即任何一个字符的编码都不是另一个字符编码的前缀。

哈夫曼编码的解码 

void HuffmanDecode(HuffmanTree HT, int n, char *code) {
    int p = 2 * n - 1;  // 根结点位置
    int i = 0;

    while (code[i] != '\0') {
        if (code[i] == '0') {
            p = HT[p].lchild;
        } else {
            p = HT[p].rchild;
        }

        if (HT[p].lchild == 0 && HT[p].rchild == 0) {  // 到达叶子结点
            printf("%c", p + 'A' - 1);  // 假设叶子结点对应字符A, B, C...
            p = 2 * n - 1;              // 回到根结点
        }
        i++;
    }
}

哈夫曼树的应用

数据压缩

哈夫曼编码广泛应用于数据压缩领域，如ZIP、GZIP等压缩算法。

文件存储

在文件系统中，哈夫曼编码用于减少存储空间。

通信系统

在数字通信中，哈夫曼编码用于提高传输效率。

图像处理

在JPEG等图像压缩标准中，哈夫曼编码被用于熵编码阶段。

哈夫曼算法的复杂度分析

操作 时间复杂度 空间复杂度
构建哈夫曼树 O(n log n) O(n)
生成哈夫曼编码 O(n²) O(n²)
编码 O(m) O(m)
解码 O(m) O(1)

其中，n是字符集大小，m是待编码字符串长度。

哈夫曼树的优缺点

优点

  • 压缩效率高
  • 编码和解码算法简单
  • 无损压缩
  • 自适应性强

缺点

  • 需要预先知道字符频率
  • 编码表需要额外存储
  • 对于小文件，压缩效果不明显
