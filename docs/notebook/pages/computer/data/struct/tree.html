树与图

树的基本概念

树（Tree） 是一种非线性的层次数据结构，由节点（Node）和边（Edge）组成。树有一个特殊的节点称为根节点，其他节点可以分为多个不相交的子树。

树的基本术语

  • 节点（Node）：树的基本单位，包含数据和指向子节点的指针
  • 根节点（Root）：没有父节点的节点，是树的起点
  • 叶节点（Leaf）：没有子节点的节点
  • 父节点（Parent）：有子节点的节点
  • 子节点（Child）：从属于父节点的节点
  • 兄弟节点（Sibling）：具有相同父节点的节点
  • 深度（Depth）：从根节点到该节点的路径长度
  • 高度（Height）：从该节点到最远叶节点的路径长度

二叉树

二叉树（Binary Tree） 是每个节点最多有两个子节点的树结构，两个子节点分别称为左子节点和右子节点。

二叉树的类型

  • 满二叉树：所有非叶节点都有两个子节点，且所有叶节点都在同一层
  • 完全二叉树：除了最后一层外，其他层都是满的，且最后一层的节点都靠左排列
  • 二叉搜索树（BST）：左子树的所有节点值小于根节点，右子树的所有节点值大于根节点
  • 平衡二叉树（AVL）：任何节点的左右子树高度差不超过1
  • 红黑树：一种自平衡二叉搜索树，确保最长路径不超过最短路径的两倍

树的遍历方式

  • 前序遍历：根节点 → 左子树 → 右子树
  • 中序遍历：左子树 → 根节点 → 右子树
  • 后序遍历：左子树 → 右子树 → 根节点
  • 层序遍历：按层次从上到下、从左到右遍历

图的基本概念

图（Graph） 是一种非线性的数据结构，由顶点（Vertex）和边（Edge）组成。图可以表示对象之间的多对多关系。

图的基本术语

  • 顶点（Vertex）：图中的基本单位，也称为节点
  • 边（Edge）：连接两个顶点的线，可以是有向或无向的
  • 度（Degree）：与顶点相连的边的数量
  • 路径（Path）：从一个顶点到另一个顶点的顶点序列
  • 环（Cycle）：起点和终点相同的路径
  • 连通图：任意两个顶点之间都有路径的图

图的类型

  • 无向图：边没有方向的图
  • 有向图：边有方向的图
  • 加权图：边带有权值的图
  • 完全图：每对顶点之间都有边相连的图
  • 稀疏图：边数远小于顶点数平方的图
  • 稠密图：边数接近顶点数平方的图

图的表示方法

  • 邻接矩阵：使用二维数组表示图中顶点之间的连接关系
  • 邻接表：为每个顶点维护一个列表，存储与该顶点相邻的顶点
  • 边列表：存储图中所有的边

图的遍历算法

  • 深度优先搜索（DFS）：沿着一条路径尽可能深入地访问顶点
  • 广度优先搜索（BFS）：按层次访问顶点，先访问距离起点近的顶点

树与图的应用

树的应用：

  • 文件系统的目录结构
  • 数据库索引（B树、B+树）
  • 编译器的语法分析
  • 决策树算法
  • 哈夫曼编码

图的应用：

  • 社交网络分析
  • 路径规划（最短路径算法）
  • 网络拓扑结构
  • 依赖关系分析
  • 推荐系统
