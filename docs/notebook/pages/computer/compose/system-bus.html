系统总线

系统总线概述

系统总线（System Bus）是连接计算机各功能部件的公共通信线路，是计算机系统的重要组成部分。它负责在各部件之间传输数据、地址和控制信号，协调整个系统的有序运行。

系统总线的功能

1. 数据传输功能

数据类型

  • 指令数据：从内存到CPU的指令传输
  • 操作数据：CPU与内存、I/O设备间的数据传输
  • 状态数据：设备状态信息的传输
  • 控制数据：控制命令和参数的传输

传输方式

  • 并行传输：多位数据同时传输
  • 串行传输：数据位按顺序传输
  • 同步传输：按统一时钟信号传输
  • 异步传输：通过握手协议传输

2. 地址传输功能

地址类型

  • 内存地址：指定内存单元的地址
  • I/O地址：指定I/O设备的地址
  • 设备地址：指定设备的标识地址

地址空间

  • 统一编址：内存和I/O设备统一编址
  • 独立编址：内存和I/O设备分别编址

3. 控制传输功能

控制信号类型

  • 时序控制：协调各部件的时序关系
  • 操作控制：控制各部件的操作
  • 状态控制：传输设备状态信息
  • 中断控制：处理中断请求和响应

控制方式

  • 集中式控制：由中央控制器统一控制
  • 分布式控制：各部件自主协调控制

系统总线的组成

1. 数据总线（Data Bus）

功能

  • 双向传输：支持数据的双向传输
  • 并行传输：多位数据同时传输
  • 缓冲功能：提供数据传输缓冲

特性

  • 宽度：决定一次传输的数据位数
  • 频率：决定数据传输速率
  • 负载能力：决定可连接的设备数量

2. 地址总线（Address Bus）

功能

  • 单向传输：通常从CPU向内存或I/O设备传输
  • 地址指定：指定内存单元或I/O端口地址
  • 地址译码：选择特定的内存单元或设备

特性

  • 宽度：决定可寻址的地址空间大小
  • n位地址总线：可寻址2^n个地址单元

3. 控制总线（Control Bus）

功能

  • 时序控制：提供系统时序信号
  • 操作控制：控制各部件的操作
  • 状态传输：传输设备状态信息
  • 中断处理：处理中断请求和响应

典型控制信号

信号名称 功能 方向
CLK 系统时钟 CPU输出
RESET 系统复位 CPU输出
READ 读使能 CPU输出
WRITE 写使能 CPU输出
READY 就绪信号 设备输出
INT 中断请求 设备输出
INTA 中断响应 CPU输出

系统总线的结构

1. 单总线结构

结构特点

  • 单一总线：所有部件共享同一总线
  • 结构简单：设计和实现相对简单
  • 成本低廉：硬件成本较低

缺点

  • 总线冲突：多个部件同时竞争总线
  • 性能瓶颈：总线成为系统性能瓶颈
  • 扩展困难：增加设备会降低性能

应用

  • 简单系统：小型嵌入式系统
  • 教学系统：计算机组成原理教学

2. 双总线结构

结构特点

  • 内存总线：CPU与内存之间的专用总线
  • I/O总线：CPU与I/O设备之间的总线
  • 分离传输：内存访问和I/O操作分离

优点

  • 减少冲突：内存和I/O操作不相互干扰
  • 提高性能：并行进行内存和I/O操作
  • 便于扩展：可以独立扩展内存或I/O

3. 三总线结构

结构特点

  • 内存总线：CPU与主存之间的总线
  • I/O总线：CPU与I/O设备之间的总线
  • DMA总线：内存与I/O设备之间的直接总线

优点

  • DMA传输：支持直接内存访问
  • 并行操作：CPU、内存、I/O可并行工作
  • 高性能：最大化系统吞吐量

缺点

  • 结构复杂：设计和控制复杂
  • 成本较高：硬件成本增加

总线仲裁机制

1. 集中式仲裁

链式查询方式

  • 总线忙：BS信号表示总线是否被占用
  • 总线请求：BR信号表示设备请求总线
  • 总线授权：BG信号依次传递授权

计数器定时查询方式

  • 计数器：中央仲裁器维护设备计数
  • 查询机制：按计数器值查询设备请求
  • 优先级：可通过计数器初始值设置优先级

独立请求方式

  • 独立请求线：每个设备有独立的请求线
  • 独立授权线：每个设备有独立的授权线
  • 优先级排队：中央仲裁器进行优先级排队

2. 分布式仲裁

自举分布式仲裁

  • 设备优先级：每个设备有固定的优先级
  • 仲裁逻辑：各设备根据优先级自主仲裁
  • 无需中央仲裁器：仲裁逻辑分布在各设备中

冲突检测分布式仲裁

  • 冲突检测：检测多个设备同时请求总线
  • 重试机制：冲突时设备重新请求
  • 随机退避：减少再次冲突的概率

总线定时方式

1. 同步定时

特点

  • 统一时钟：所有设备使用统一的时钟信号
  • 固定时序：操作按固定时序进行
  • 简单控制：控制逻辑相对简单

优点

  • 速度快：适合高速设备
  • 控制简单：时序控制逻辑简单
  • 成本低：硬件实现成本低

缺点

  • 时钟偏移：高速时钟可能产生偏移
  • 速度限制：受最慢设备限制
  • 灵活性差：不适合速度差异大的设备

2. 异步定时

特点

  • 握手协议：使用握手协议协调操作
  • 无时钟：不依赖统一时钟信号
  • 自适应：适应不同速度的设备

握手过程

  1. 请求：主设备发出操作请求
  2. 应答：从设备回应操作准备就绪
  3. 数据传输：进行数据传输
  4. 完成：从设备通知操作完成

优点

  • 适应性强：适合不同速度的设备
  • 无时钟偏移：避免时钟偏移问题
  • 功耗低：不需要全局时钟

缺点

  • 速度慢：握手过程增加延迟
  • 控制复杂：握手逻辑复杂
  • 成本高：硬件实现成本较高

3. 半同步定时

特点

  • 结合优势：结合同步和异步的优势
  • 灵活时序：提供灵活的时序控制
  • 等待状态：支持插入等待状态

应用场景

  • 混合系统：包含不同速度设备的系统
  • 扩展系统：需要支持慢速扩展设备的系统

总线标准

1. ISA总线

特点

  • 8/16位数据总线：支持8位或16位数据传输
  • 24位地址总线：可寻址16MB地址空间
  • 8MHz时钟：标准工作频率8MHz

应用

  • 早期PC：IBM PC/AT及其兼容机
  • 扩展卡：声卡、网卡等扩展设备

2. PCI总线

特点

  • 32/64位数据总线：支持32位或64位数据传输
  • 33/66MHz时钟：支持33MHz或66MHz工作频率
  • 即插即用：支持即插即用功能
  • 突发传输：支持突发数据传输模式

架构

  • 主桥：连接CPU和PCI总线
  • PCI设备：各种PCI接口设备
  • PCI桥：连接多个PCI总线段

3. PCI Express总线

特点

  • 串行传输：采用高速串行传输
  • 点对点连接：设备间点对点连接
  • 高带宽：单通道带宽可达2.5Gbps（PCIe 1.0）
  • 可扩展：支持x1、x4、x8、x16等多种通道配置

架构

  • 根复合体：连接CPU和PCIe设备
  • 交换机：扩展PCIe端口
  • 端点设备：各种PCIe接口设备

4. USB总线

特点

  • 通用串行总线：支持多种设备类型
  • 热插拔：支持设备热插拔
  • 即插即用：自动识别和配置设备
  • 供电能力：可为设备提供电源

版本演进

版本 发布年份 最大速率 特点

总线性能分析

1. 带宽计算

基本公式

总线带宽 = 总线宽度 × 时钟频率 × 传输效率

示例计算

  • 32位总线，33MHz：32 × 33 MHz = 1056 Mbps = 132 MB/s
  • 64位总线，66MHz：64 × 66 MHz = 4224 Mbps = 528 MB/s

2. 性能影响因素

总线宽度

  • 数据宽度：影响单次传输的数据量
  • 地址宽度：影响可寻址空间大小

总线频率

  • 时钟频率：决定总线的工作速度
  • 倍频技术：通过倍频提高有效频率

传输方式

  • 突发传输：连续传输多个数据
  • 块传输：传输数据块
  • 流水线传输：重叠多个传输操作

3. 性能优化技术

总线缓冲

  • 数据缓冲：减少总线占用时间
  • 地址缓冲：提前准备地址信息

总线复用

  • 地址/数据复用：同一组线传输地址和数据
  • 控制复用：多个控制信号共享线路

总线分级

  • 局部总线：高速设备专用总线
  • 系统总线：连接主要功能部件
  • 扩展总线：连接低速扩展设备
