补码运算

补码表示法概述

补码（Two's Complement）是计算机中表示有符号整数的标准方法。它解决了原码和反码表示中存在的+0和-0问题，并使得加减法运算统一。

补码的定义与性质

补码定义

对于n位二进制数，其补码定义为：

  • 正数：与原码相同
  • 负数：对应正数的补码 = 2^n - |x|

补码求法

求一个负数的补码有两种方法：

  1. 定义法：2^n - |x|
  2. 简便法：原码取反加1

例：求-5的8位补码表示

方法1：2^8 - 5 = 256 - 5 = 251 = 11111011₂

方法2：+5的原码 = 00000101，取反 = 11111010，加1 = 11111011

补码表示范围

对于n位补码：

  • 表示范围：-2^(n-1) 到 2^(n-1)-1
  • 零的表示：唯一（全0）
  • 符号位：最高位为1表示负数，为0表示正数

补码运算规则

加法运算

补码加法直接将两个数的补码相加，忽略最高位的进位。

例：5 + (-3) = ?

+5的补码：00000101

-3的补码：11111101

相加：00000101 + 11111101 = 100000010

忽略进位：00000010 = +2

减法运算

补码减法转换为加法：A - B = A + (-B)

例：5 - 3 = 5 + (-3) = ?

+5的补码：00000101

-3的补码：11111101

相加：00000101 + 11111101 = 100000010

结果：00000010 = +2

溢出判断

当两个同号数相加结果符号相反时发生溢出。

  • 正溢出：两个正数相加得负数
  • 负溢出：两个负数相加得正数

例：4位补码，7 + 1 = ?

7的补码：0111

1的补码：0001

相加：0111 + 0001 = 1000 = -8（溢出！）

补码乘法运算

布斯算法（Booth's Algorithm）

布斯算法是补码乘法的有效方法，通过检测乘数位模式减少部分积数量。

算法步骤

  1. 初始化：A=0，Q=乘数，M=被乘数，Q₋₁=0
  2. 检查Q₀Q₋₁：
    • 00或11：算术右移
    • 01：A = A + M，然后算术右移
    • 10：A = A - M，然后算术右移
  3. 重复n次（n为位数）

布斯算法示例

例：计算3 × (-2)，4位表示

M = 0011 (+3)，-M = 1101 (-3)

Q = 1110 (-2)

初始：A=0000，Q=1110，Q₋₁=0

过程略，最终结果为：11111010 = -6₁₀

补码除法运算

恢复余数法

  1. 初始化：被除数扩展符号位，除数取补码
  2. 试商：被除数减去除数
  3. 判断余数符号：
    • 余数为正：商1，不恢复
    • 余数为负：商0，恢复余数
  4. 左移，重复直到完成

不恢复余数法

改进的除法算法，避免恢复余数操作，提高效率。

补码运算的硬件实现

加法器设计

  • 半加器：处理两个1位二进制数相加
  • 全加器：处理三个1位二进制数相加（含进位）
  • 行波进位加法器：级联全加器实现多位加法
  • 超前进位加法器：减少进位延迟，提高速度

溢出检测电路

溢出 = Cₙ ⊕ Cₙ₋₁，其中Cₙ为符号位进位，Cₙ₋₁为最高数值位进位。

补码运算的应用

有符号整数运算

  • C/C++、Java等语言的有符号整数使用补码
  • 统一了加减法运算
  • 简化了比较运算

地址计算

  • 数组索引计算
  • 指针运算
  • 栈操作

补码运算的注意事项

符号扩展

将短字长补码扩展为长字长时，符号位需要扩展。

例：8位补码11111011（-5）扩展为16位

结果：1111111111111011

截断误差

将长字长补码截断为短字长时可能产生误差。

运算优先级

  • 乘除法优先于加减法
  • 注意运算过程中的溢出
  • 考虑运算结果的表示范围
