虚拟存储器

虚拟存储器概述

虚拟存储器（Virtual Memory）是一种存储管理技术，它为每个进程提供一个独立的、连续的地址空间，使得程序可以访问比实际物理内存更大的存储空间。虚拟存储器通过硬件和操作系统的配合，实现了内存的抽象和扩展。

虚拟存储器的基本概念

1. 虚拟地址空间

  • 逻辑地址：程序使用的地址，也称为虚拟地址
  • 物理地址：实际内存中的地址
  • 地址转换：将虚拟地址映射到物理地址

2. 地址空间划分

  • 用户空间：用户程序运行的地址空间
  • 内核空间：操作系统内核使用的地址空间
  • 共享空间：多个进程共享的内存区域

3. 虚拟存储器的优势

  • 内存扩展：程序可以使用比物理内存更大的地址空间
  • 内存保护：每个进程有独立的地址空间，防止相互干扰
  • 内存共享：多个进程可以共享相同的代码和数据
  • 简化编程：程序员无需关心物理内存布局

虚拟存储器的实现技术

1. 分页（Paging）

基本概念

  • 页面（Page）：虚拟地址空间的基本单位，通常为4KB
  • 页框（Page Frame）：物理内存中的页面容器
  • 页表（Page Table）：记录虚拟页面到物理页框的映射关系

地址转换过程

  1. CPU生成虚拟地址
  2. MMU（内存管理单元）将虚拟地址分解为页号和页内偏移
  3. 查询页表获取物理页框号
  4. 组合物理页框号和页内偏移得到物理地址

页表结构

  • 单级页表：简单但占用大量连续内存
  • 多级页表：节省内存空间，如二级、三级页表
  • 反向页表：每个物理页框对应一个表项，节省空间

2. 分段（Segmentation）

基本概念

  • 段（Segment）：逻辑上的程序和数据单元
  • 段表（Segment Table）：记录段的起始地址和长度
  • 逻辑结构：代码段、数据段、堆栈段等

地址转换过程

  1. 虚拟地址分为段号和段内偏移
  2. 查询段表获取段基址和段长度
  3. 检查段内偏移是否越界
  4. 段基址加段内偏移得到物理地址

3. 段页式（Segmented Paging）

结合分段和分页的优点，先分段再分页。

  • 将程序分为逻辑段
  • 每个段内采用分页管理
  • 兼具灵活性和效率

页面置换算法

1. 最佳置换（OPT）

置换将来最长时间不会被访问的页面，理论最优但无法实现。

  • 作为性能评价基准
  • 实际系统中无法预知未来访问

2. 先进先出（FIFO）

置换最早进入内存的页面。

  • 实现简单，使用队列结构
  • 可能出现Belady异常
  • 性能一般

3. 最近最少使用（LRU）

置换最长时间未被访问的页面。

  • 符合局部性原理，性能较好
  • 硬件实现复杂，需要维护访问时间
  • 近似算法如Clock算法更实用

4. 时钟算法（Clock）

使用环形链表和引用位，近似实现LRU。

  • 硬件实现简单
  • 性能接近LRU
  • 广泛应用于实际系统

5. 最不经常使用（LFU）

置换访问次数最少的页面。

  • 需要维护访问计数
  • 对程序行为变化适应性差

虚拟存储器管理

1. 地址转换机制

TLB（转换后备缓冲器）

  • 功能：缓存最近使用的页表项，加速地址转换
  • 结构：类似于Cache的相联存储器
  • 命中率：通常高达90%以上

MMU（内存管理单元）

  • 功能：执行虚拟地址到物理地址的转换
  • 组成：TLB、页表遍历单元、权限检查单元
  • 异常处理：处理页缺失、权限违规等异常

2. 页面分配策略

固定分配

  • 每个进程分配固定数量的页框
  • 实现简单，但不够灵活

可变分配

  • 根据进程需要动态分配页框
  • 更有效地利用内存资源
  • 需要全局页面置换策略

3. 页面置换策略

局部置换

  • 只置换本进程的页面
  • 进程之间相互独立

全局置换

  • 可以从所有进程中选择页面置换
  • 更好地利用整个系统内存
  • 可能导致进程间干扰

虚拟存储器性能

1. 性能指标

  • 页缺失率：页面访问中发生缺失的比例
  • 有效访问时间：考虑页缺失惩罚的平均访问时间
  • 工作集大小：进程在一段时间内活跃访问的页面集合

2. 抖动（Thrashing）

当系统频繁进行页面置换，导致CPU大部分时间用于页面调度而非执行程序。

产生原因

  • 进程数量过多，超出物理内存容量
  • 工作集大小超过分配的物理页框数
  • 页面置换算法选择不当

解决方法

  • 减少同时运行的进程数
  • 增加物理内存
  • 改进页面置换算法
  • 使用工作集模型进行内存分配

虚拟存储器实现技术

1. 请求调页（Demand Paging）

  • 页面只有在被访问时才调入内存
  • 减少初始加载时间和内存占用
  • 需要处理页缺失异常

2. 预调页（Prepaging）

  • 在页面被访问前预先调入
  • 减少页缺失率
  • 需要预测程序行为

3. 内存映射文件

  • 将文件内容映射到虚拟地址空间
  • 简化文件I/O操作
  • 实现进程间共享内存

4. 写时复制（Copy-on-Write）

  • 多个进程共享同一物理页面
  • 写操作时创建页面副本
  • 减少内存占用，提高创建进程效率

现代虚拟存储器系统

1. Linux虚拟存储器

  • 三级页表：PGD、PMD、PTE
  • 反向映射：快速查找映射到物理页的虚拟页
  • Huge Pages：支持大页面减少TLB缺失
  • KSM：内核同页合并技术

2. Windows虚拟存储器

  • 二级页表：页目录和页表
  • 工作集管理：基于工作集的页面置换
  • 超级页面：支持大页面
  • 地址窗口扩展：扩展32位系统的寻址能力

3. 硬件支持

  • Intel VT-x：硬件虚拟化支持
  • AMD-V：AMD的虚拟化技术
  • EPT/NPT：扩展页表/嵌套页表

虚拟存储器发展趋势

1. 大容量支持

  • 64位系统：支持更大的虚拟地址空间
  • 大页面：减少页表开销和TLB缺失
  • 稀疏地址空间：有效利用大地址空间

2. 性能优化

  • 硬件加速：专用MMU和TLB
  • 并行处理：并行页表遍历
  • 预测技术：预取和预转换

3. 安全性增强

  • 内存保护：防止缓冲区溢出攻击
  • 地址空间布局随机化：ASLR技术
  • 控制流完整性：防止ROP攻击
