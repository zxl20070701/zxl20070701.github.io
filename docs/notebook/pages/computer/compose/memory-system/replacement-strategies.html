替换策略

替换策略概述

替换策略（Replacement Strategies）是存储层次结构中当新数据需要存储而目标位置已满时，决定替换哪些现有数据的算法。替换策略直接影响系统性能，是存储管理的关键组成部分。

替换策略的基本原理

1. 局部性原理应用

  • 时间局部性：最近访问的数据可能再次被访问
  • 空间局部性：相邻数据可能一起被访问
  • 预测未来：基于历史访问模式预测未来访问

2. 性能评价指标

  • 命中率：成功找到数据的概率
  • 缺失率：1 - 命中率
  • 缺失惩罚：替换操作的时间开销
  • 公平性：各进程获得合理的资源份额

Cache替换策略

1. 先进先出（FIFO）

基本原理

替换最早进入Cache的数据块，使用队列结构实现。

实现机制

  • 时间戳：记录每个数据块的进入时间
  • 队列结构：新数据入队，替换数据出队
  • 循环计数器：使用计数器实现队列功能

优缺点分析

  • 优点：实现简单，硬件开销小
  • 缺点：不考虑访问频率，可能替换频繁使用的数据
  • 改进：Second Chance算法增加引用位

2. 最近最少使用（LRU）

基本原理

替换最长时间未被访问的数据块，符合时间局部性原理。

实现机制

  • 计数器法：为每个数据块维护访问时间戳
  • 栈结构：访问时将数据移到栈顶
  • 位图法：使用访问位矩阵记录访问历史

近似LRU算法

  • Clock算法：使用循环链表和引用位
  • NFU算法：非频繁使用算法，定期衰减计数器
  • LRU-K：考虑最近K次访问的时间

优缺点分析

  • 优点：符合局部性原理，命中率较高
  • 缺点：硬件实现复杂，需要维护访问历史
  • 应用：广泛应用于现代处理器Cache

3. 随机替换（Random）

基本原理

随机选择一个数据块进行替换，无需维护历史信息。

实现机制

  • 随机数生成器：生成随机索引
  • 伪随机：使用线性反馈移位寄存器
  • 哈希函数：基于当前状态生成随机选择

优缺点分析

  • 优点：实现最简单，硬件开销最小
  • 缺点：性能不稳定，可能表现很好也可能很差
  • 应用：简单系统或作为基准算法

4. 最不经常使用（LFU）

基本原理

替换访问次数最少的数据块，基于频率而非时间。

实现机制

  • 计数器数组：记录每个数据块的访问次数
  • 定期衰减：防止历史访问影响过大
  • 分层计数：使用多级计数器减少开销

优缺点分析

  • 优点：考虑长期访问模式
  • 缺点：对程序行为变化适应性差，需要维护计数器
  • 改进：结合LRU和LFU的混合算法

虚拟存储器替换策略

1. 最佳置换（OPT）

基本原理

置换将来最长时间不会被访问的页面，理论最优算法。

特点分析

  • 优点：提供性能上界，作为评价基准
  • 缺点：需要预知未来，无法实现
  • 应用：算法分析和性能评估

2. 先进先出（FIFO）

基本原理

替换最早进入内存的页面，使用队列结构。

Belady异常

  • 现象：增加页面数反而降低命中率
  • 原因：FIFO不考虑访问模式
  • 影响：在某些访问模式下出现

改进算法

  • Second Chance：给页面第二次机会
  • Clock算法：循环链表实现

3. 最近最少使用（LRU）

基本原理

替换最长时间未被访问的页面，基于时间局部性。

实现挑战

  • 时间戳维护：需要精确记录每次访问时间
  • 硬件开销：需要额外的存储和逻辑
  • 近似实现：使用访问位和老化机制

近似算法

  • LRU Clock：使用循环链表
  • NFU：非频繁使用算法
  • Aging：老化算法，定期右移计数器

4. 时钟算法（Clock）

基本原理

使用循环链表和引用位，近似实现LRU算法。

实现机制

  • 循环链表：所有页面组成环形结构
  • 时钟指针：指示当前检查位置
  • 引用位：记录页面是否被访问

算法步骤

  1. 检查时钟指针指向的页面
  2. 如果引用位为0，选择该页面替换
  3. 如果引用位为1，清零引用位，指针前进
  4. 重复直到找到可替换页面

优缺点分析

  • 优点：实现简单，性能接近LRU
  • 缺点：可能需要多次扫描
  • 应用：广泛应用于实际操作系统

工作集模型

1. 基本概念

  • 工作集：进程在一段时间内活跃访问的页面集合
  • 工作集大小：工作集中的页面数量
  • 时间窗口：定义工作集的时间范围

2. 工作集算法

基本原理

只保留进程的工作集在内存中，其余页面可以置换。

实现机制

  • 时间戳维护：记录每个页面的最后访问时间
  • 窗口大小：通常设置为几十个页面访问
  • 动态调整：根据工作集大小调整内存分配

优缺点分析

  • 优点：理论上最优，防止抖动
  • 缺点：实现复杂，需要维护时间信息
  • 应用：作为理论模型和算法基准

页面分配策略

1. 固定分配

基本原理

每个进程分配固定数量的页框，不随执行情况变化。

优缺点分析

  • 优点：实现简单，内存管理容易
  • 缺点：不够灵活，可能造成资源浪费或不足
  • 应用：简单系统或实时系统

2. 可变分配

基本原理

根据进程需要动态调整分配的页框数量。

实现机制

  • 工作集监测：跟踪进程的工作集大小
  • 缺页率反馈：根据缺页率调整分配
  • 全局平衡：在进程间合理分配内存资源

优缺点分析

  • 优点：更有效地利用内存资源
  • 缺点：实现复杂，需要全局管理策略
  • 应用：现代操作系统的主要策略

全局替换策略

1. 局部置换 vs 全局置换

局部置换

  • 只替换本进程的页面
  • 进程间相互独立，无干扰
  • 可能导致内存利用不充分

全局置换

  • 可以从所有进程中选择页面替换
  • 更好地利用整个系统内存
  • 可能导致进程间干扰

2. 页面置换范围

  • 局部范围：只考虑当前进程的页面
  • 全局范围：考虑系统中所有进程的页面
  • 混合策略：结合局部和全局的优点

现代替换策略

1. Linux页面替换

双链表算法

  • 活跃链表：最近访问的页面
  • 非活跃链表：较长时间未访问的页面
  • 动态平衡：根据系统负载调整链表大小

特点

  • 近似LRU算法
  • 考虑页面类型（匿名页、文件页）
  • 支持内存回收和压缩

2. Windows页面替换

工作集管理

  • 工作集调整：动态调整每个进程的工作集
  • 缺页率反馈：根据缺页率调整内存分配
  • 平衡集管理：在内存紧张时回收页面

特点

  • 基于工作集模型
  • 考虑进程优先级
  • 支持超级页面

替换策略性能分析

1. 评价方法

  • 模拟实验：使用实际程序trace进行模拟
  • 理论分析：基于数学模型分析性能
  • 实际测量：在真实系统中测量性能

2. 影响因素

  • 程序行为：访问模式、工作集大小
  • 系统配置：内存大小、页面大小
  • 负载特征：并发进程数、内存压力
