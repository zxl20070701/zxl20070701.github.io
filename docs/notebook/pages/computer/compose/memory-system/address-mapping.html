地址映射

地址映射概述

地址映射（Address Mapping）是计算机系统中将逻辑地址转换为物理地址的过程。它是存储管理的核心机制，使得程序可以使用连续的虚拟地址空间，而无需关心物理内存的实际布局。

地址映射的基本概念

1. 地址类型

  • 逻辑地址：程序使用的地址，也称为虚拟地址
  • 物理地址：实际内存中的地址
  • 线性地址：分段机制中的中间地址

2. 地址空间

  • 虚拟地址空间：程序可见的地址范围
  • 物理地址空间：实际内存的地址范围
  • 地址空间大小：由处理器架构决定

3. 映射机制

  • 硬件支持：MMU（内存管理单元）完成地址转换
  • 软件支持：操作系统维护映射表
  • 透明性：对应用程序透明

分页地址映射

1. 分页基本概念

  • 页面（Page）：虚拟地址空间的基本单位
  • 页框（Page Frame）：物理内存中的页面容器
  • 页大小：通常为4KB，也有2MB、1GB等大页

2. 地址结构

虚拟地址结构

  • 页号：标识虚拟页面
  • 页内偏移：页面内的字节偏移

物理地址结构

  • 页框号：标识物理页框
  • 页内偏移：与虚拟地址的页内偏移相同

3. 页表机制

页表项（PTE）

  • 页框号：存储物理页框号
  • 有效位：指示该页是否在内存中
  • 权限位：读/写/执行权限
  • 脏位：指示页面是否被修改
  • 访问位：指示页面是否被访问

地址转换过程

  1. 提取虚拟地址的页号和页内偏移
  2. 查询页表获取页框号
  3. 组合页框号和页内偏移得到物理地址

4. 多级页表

二级页表

  • 页目录：存储页表的物理地址
  • 页表：存储页框号和相关控制位
  • 优势：节省内存空间，支持稀疏地址空间

三级页表

  • 页目录指针表：指向页目录
  • 页目录：指向页表
  • 页表：存储页框信息
  • 应用：64位系统，如x86-64

分段地址映射

1. 分段基本概念

  • 段（Segment）：逻辑上的程序和数据单元
  • 段表：记录段的起始地址和长度
  • 段大小：可变，通常从1字节到4GB

2. 地址结构

虚拟地址结构

  • 段选择子：标识段描述符
  • 段内偏移：段内的字节偏移

3. 段表机制

段描述符

  • 基地址：段在物理内存中的起始地址
  • 段界限：段的最大长度
  • 访问权限：读/写/执行权限
  • 段类型：代码段、数据段、系统段等

地址转换过程

  1. 使用段选择子定位段描述符
  2. 检查段内偏移是否越界
  3. 段基地址加段内偏移得到线性地址

段页式地址映射

1. 基本概念

结合分段和分页的优点，先分段再分页。

  • 逻辑结构：程序分为逻辑段
  • 物理实现：每个段内采用分页管理
  • 优势：兼具灵活性和效率

2. 地址转换过程

  1. 虚拟地址分为段选择子和段内偏移
  2. 通过段表获取段基址
  3. 段基址加段内偏移得到线性地址
  4. 线性地址通过页表转换为物理地址

3. 实际应用

  • x86架构：支持段页式管理
  • 现代操作系统：主要使用分页，分段用于特殊用途

TLB（转换后备缓冲器）

1. TLB基本概念

  • 功能：缓存最近使用的页表项
  • 位置：MMU内部
  • 作用：加速地址转换过程

2. TLB结构

TLB项

  • 虚拟页号：作为查找键
  • 物理页框号：转换结果
  • 控制位：有效位、脏位、权限位等

映射方式

  • 全相联：任何虚拟页号可映射到任何TLB项
  • 组相联：虚拟页号映射到特定组的项

3. TLB操作

TLB命中

  • 直接获取物理页框号
  • 组合页框号和页内偏移得到物理地址
  • 无需访问页表，速度极快

TLB缺失

  • 查询页表获取映射信息
  • 更新TLB项
  • 如果页表项无效，触发页缺失异常

4. TLB性能

  • 命中率：通常高于99%
  • 缺失惩罚：几十到几百个时钟周期
  • 优化：多级TLB、大页支持

大页映射

1. 大页基本概念

  • 标准页：通常为4KB
  • 大页：2MB、1GB甚至更大
  • 优势：减少页表大小，提高TLB效率

2. 大页应用

  • 数据库系统：大内存工作集
  • 科学计算：大数组和矩阵
  • 虚拟化：减少嵌套页表开销

3. 大页管理

  • 分配策略：连续物理内存分配
  • 碎片问题：可能产生内存碎片
  • 透明大页：操作系统自动管理大页

反向映射

1. 基本概念

  • 功能：从物理页框查找映射的虚拟页
  • 应用：页面回收、页面共享、内存压缩
  • 挑战：需要维护额外的数据结构

2. 实现机制

基于对象的反向映射

  • 结构：每个物理页维护映射关系列表
  • 优势：查找速度快
  • 劣势：内存开销大

基于搜索的反向映射

  • 方法：遍历所有进程的页表
  • 优势：内存开销小
  • 劣势：查找速度慢

多级地址映射

1. 虚拟化环境

  • 客户虚拟地址：虚拟机内的虚拟地址
  • 客户物理地址：虚拟机看到的物理地址
  • 主机物理地址：实际的物理内存地址

2. 嵌套页表

Intel EPT（扩展页表）

  • 功能：客户物理地址到主机物理地址的映射
  • 加速：硬件支持，减少VMM干预
  • TLB：支持嵌套TLB，提高转换效率

AMD NPT（嵌套页表）

  • 功能：与EPT类似
  • 实现：硬件支持地址转换
  • 性能：接近原生性能

地址映射优化

1. 性能优化

  • TLB优化：多级TLB、大页支持
  • 页表优化：多级页表、哈希页表
  • 预取技术：预取页表项

2. 安全优化

  • ASLR：地址空间布局随机化
  • NX位：禁止执行保护
  • SMEP/SMAP：管理模式访问保护

3. 功耗优化

  • 动态TLB管理：根据负载调整TLB大小
  • 页表压缩：减少内存访问
  • 低功耗模式：智能电源管理

现代处理器地址映射实例

1. x86-64架构

  • 虚拟地址：48位（256TB地址空间）
  • 物理地址：52位（4PB物理内存）
  • 页表：四级页表（PML4、PDPT、PD、PT）
  • 大页支持：2MB、1GB大页

2. ARMv8架构

  • 虚拟地址：48位
  • 物理地址：48位（可扩展至52位）
  • 页表：四级页表
  • 大页支持：64KB、2MB、32MB、512MB、1GB

3. RISC-V架构

  • 虚拟地址：39位或48位
  • 物理地址：32位到56位
  • 页表：三级或四级页表
  • 特点：模块化设计，可配置
