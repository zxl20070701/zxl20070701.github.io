Cache

Cache存储器概述

Cache（高速缓冲存储器）是位于CPU和主存之间的小容量、高速度的存储器，用于解决CPU和主存之间速度不匹配的问题。Cache利用程序访问的局部性原理，通过存储最近使用的数据和指令来提高系统性能。

局部性原理

1. 时间局部性

最近被访问的信息很可能在不久的将来再次被访问。

  • 循环结构中的指令重复执行
  • 变量的重复访问
  • 栈操作中的数据重用

2. 空间局部性

与当前被访问信息地址相近的信息很可能被访问。

  • 数组元素的顺序访问
  • 指令的顺序执行
  • 数据结构的连续存储

Cache的基本结构

1. Cache的组成

  • 存储阵列：存储数据和指令
  • 标记存储器：存储地址标记信息
  • 有效位：指示Cache行是否有效
  • 控制逻辑：管理Cache的读写操作

2. Cache的层次结构

  • L1 Cache：一级缓存，速度最快，容量最小（通常16-64KB）
  • L2 Cache：二级缓存，速度和容量居中（通常256KB-2MB）
  • L3 Cache：三级缓存，速度较慢，容量最大（通常2-32MB）

Cache映射方式

1. 直接映射（Direct Mapping）

主存中的每个块只能映射到Cache中的一个特定位置。

地址格式

  • 标记（Tag）：标识主存块
  • 索引（Index）：选择Cache行
  • 偏移（Offset）：选择块内字节

特点

  • 硬件实现简单，成本低
  • 冲突率高，容易发生抖动
  • 适用于大容量Cache

2. 全相联映射（Fully Associative Mapping）

主存中的任何块都可以映射到Cache中的任何位置。

地址格式

  • 标记：标识主存块
  • 偏移：选择块内字节

特点

  • 冲突率最低，空间利用率最高
  • 硬件复杂，成本高
  • 需要并行比较所有标记
  • 适用于小容量Cache

3. 组相联映射（Set Associative Mapping）

将Cache分成若干组，每组包含多个行，主存块可以映射到特定组的任意行。

地址格式

  • 标记：标识主存块
  • 组索引：选择Cache组
  • 偏移：选择块内字节

特点

  • 折中方案，平衡了硬件复杂度和性能
  • 2路、4路、8路组相联最常见
  • 广泛应用于现代处理器

Cache替换策略

1. 先进先出（FIFO）

替换最早进入Cache的块。

  • 实现简单，使用队列结构
  • 不考虑访问频率，可能替换频繁使用的块

2. 最近最少使用（LRU）

替换最长时间未被访问的块。

  • 符合局部性原理，性能较好
  • 硬件实现复杂，需要记录访问历史
  • 近似LRU算法在实际中更常用

3. 随机替换（Random）

随机选择一个块进行替换。

  • 实现最简单，硬件开销最小
  • 性能不稳定，但在某些情况下表现良好

4. 最不经常使用（LFU）

替换访问次数最少的块。

  • 需要维护访问计数器
  • 对程序行为变化适应性差

Cache写策略

1. 写直达（Write Through）

写操作时同时更新Cache和主存。

  • 优点：数据一致性高，实现简单
  • 缺点：写操作延迟大，占用总线带宽

2. 写回（Write Back）

写操作时只更新Cache，当Cache块被替换时才写回主存。

  • 优点：减少主存访问，提高性能
  • 缺点：数据一致性复杂，需要脏位标记

3. 写分配（Write Allocate）

写缺失时将数据块调入Cache。

  • 适用于写回策略
  • 后续写操作可以在Cache中完成

4. 非写分配（No-Write Allocate）

写缺失时直接将数据写入主存，不调入Cache。

  • 适用于写直达策略
  • 减少不必要的Cache填充

Cache性能分析

1. 性能指标

  • 命中率（Hit Rate）：访问在Cache中找到所需数据的概率
  • 缺失率（Miss Rate）：1 - 命中率
  • 命中时间（Hit Time）：Cache命中时的访问时间
  • 缺失惩罚（Miss Penalty）：Cache缺失时从主存获取数据的时间

2. 平均访问时间

平均访问时间 = 命中时间 + 缺失率 × 缺失惩罚

3. 影响性能的因素

  • Cache大小：容量越大，命中率越高
  • 块大小：块越大，空间局部性越好，但缺失惩罚增加
  • 相联度：相联度越高，冲突越少，但硬件复杂度增加
  • 替换策略：影响Cache空间利用率

多级Cache设计

1. 包含策略

  • 严格包含：L1 Cache的内容必须包含在L2 Cache中
  • 非包含：各级Cache独立，无包含关系
  • 排他：数据只能存在于某一级Cache中

2. 一致性维护

  • 监听协议：通过总线监听维护一致性
  • 目录协议：使用目录记录数据状态
  • MESI协议：Modified、Exclusive、Shared、Invalid状态

3. 预取技术

  • 顺序预取：预取连续的Cache块
  • 跨步预取：预取固定间隔的数据
  • 自适应预取：根据程序行为动态调整

Cache优化技术

1. 硬件优化

  • 流水线Cache：将Cache访问分为多个阶段
  • 非阻塞Cache：Cache缺失时不阻塞后续访问
  • 多级Cache：通过层次化设计平衡速度和容量

2. 软件优化

  • 数据布局优化：改善空间局部性
  • 循环优化：改善时间局部性
  • 分块技术：使工作集适应Cache大小

3. 编译器优化

  • 循环变换：改进循环的局部性
  • 数据预取：插入预取指令
  • 寄存器分配：减少内存访问

现代Cache设计实例

1. Intel Core处理器

  • L1 Cache：32KB指令Cache + 32KB数据Cache
  • L2 Cache：256KB，每个核心独享
  • L3 Cache：共享，容量从4MB到40MB不等
  • 映射方式：8路组相联

2. ARM Cortex处理器

  • L1 Cache：16KB-64KB指令和数据Cache
  • L2 Cache：128KB-2MB
  • L3 Cache：部分型号支持，1MB-8MB
  • 特点：低功耗设计，适合移动设备

3. GPU Cache

  • L1 Cache：每个流处理器拥有
  • L2 Cache：所有流处理器共享
  • 特点：针对图形和并行计算优化
