输入输出系统

输入输出系统概述

输入输出系统（Input/Output System）是计算机系统的重要组成部分，负责计算机与外部世界之间的信息交换。它连接各种外部设备，实现数据的输入、输出和控制，是用户与计算机交互的桥梁。

输入输出系统的功能

1. 数据交换功能

输入功能

  • 数据采集：从外部设备采集数据
  • 数据转换：将外部信号转换为计算机可识别的格式
  • 数据缓冲：临时存储输入数据
  • 数据校验：检查输入数据的正确性

输出功能

  • 数据转换：将计算机数据转换为外部设备可识别的格式
  • 数据驱动：驱动外部设备工作
  • 数据缓冲：临时存储输出数据
  • 状态反馈：获取设备的工作状态

2. 设备控制功能

设备管理

  • 设备识别：识别和配置外部设备
  • 设备初始化：初始化设备的工作状态
  • 设备调度：协调多个设备的工作
  • 设备释放：释放设备资源

错误处理

  • 错误检测：检测设备操作中的错误
  • 错误恢复：从错误状态中恢复
  • 错误报告：向系统报告错误信息

3. 接口适配功能

信号适配

  • 电平转换：转换不同的电平标准
  • 阻抗匹配：匹配不同的阻抗特性
  • 信号整形：改善信号质量

协议转换

  • 时序转换：转换不同的时序要求
  • 格式转换：转换不同的数据格式
  • 速率匹配：匹配不同的传输速率

输入输出设备的分类

1. 按数据传输方向分类

输入设备

  • 键盘：字符和命令输入
  • 鼠标：指针控制和选择
  • 扫描仪：图像输入
  • 麦克风：音频输入
  • 摄像头：视频输入

输出设备

  • 显示器：视觉信息输出
  • 打印机：文档和图像打印
  • 扬声器：音频输出
  • 绘图仪：工程图纸输出

输入输出设备

  • 触摸屏：同时支持输入和输出
  • 网络设备：双向数据通信
  • 存储设备：支持读写操作

2. 按数据传输方式分类

字符设备

  • 特点：以字符为单位传输数据
  • 例子：键盘、鼠标、串口
  • 访问方式：顺序访问，不可随机访问

块设备

  • 特点：以数据块为单位传输数据
  • 例子：硬盘、光盘、U盘
  • 访问方式：支持随机访问

3. 按设备速度分类

低速设备

  • 速度范围：几Bps到几KBps
  • 例子：键盘、鼠标、打印机

中速设备

  • 速度范围：几十KBps到几MBps
  • 例子：扫描仪、音频设备

高速设备

  • 速度范围：几十MBps到几GBps
  • 例子：硬盘、显卡、网络接口

输入输出接口

1. 接口的功能

数据缓冲

  • 速度匹配：匹配CPU与设备的速度差异
  • 数据暂存：临时存储传输数据
  • 批量传输：支持批量数据传输

信号转换

  • 并行-串行转换：转换数据格式
  • 电平转换：转换电平标准
  • 时序转换：转换时序要求

设备控制

  • 命令译码：解释CPU的控制命令
  • 状态检测：检测设备的工作状态
  • 中断控制：处理中断请求

2. 接口的组成

数据寄存器

  • 输入数据寄存器：暂存输入数据
  • 输出数据寄存器：暂存输出数据

状态寄存器

  • 设备状态：记录设备的当前状态
  • 错误状态：记录设备的错误信息

控制寄存器

  • 命令寄存器：存储控制命令
  • 模式寄存器：设置设备工作模式

地址译码器

  • 地址识别：识别设备的地址
  • 片选信号：产生设备选择信号

3. 接口的类型

并行接口

  • 特点：多位数据同时传输
  • 例子：并行打印机接口、IDE接口
  • 优点：传输速度快
  • 缺点：线路复杂，成本高

串行接口

  • 特点：数据位按顺序传输
  • 例子：串口、USB、SATA
  • 优点：线路简单，成本低
  • 缺点：传输速度相对较低

输入输出控制方式

1. 程序查询方式

工作原理

  • 轮询检测：CPU不断检测设备状态
  • 条件判断：判断设备是否就绪
  • 数据传输：就绪时进行数据传输

工作流程

  1. 发送命令：CPU向设备发送操作命令
  2. 状态查询：CPU不断查询设备状态
  3. 数据传输：设备就绪时传输数据
  4. 状态检查：检查传输是否完成

特点

优点 缺点
硬件简单 CPU效率低
控制简单 实时性差
成本低 不适合高速设备

2. 程序中断方式

工作原理

  • 中断请求：设备就绪时向CPU发送中断
  • 中断响应：CPU响应中断请求
  • 中断处理：执行中断服务程序
  • 中断返回：返回原程序继续执行

中断处理流程

  1. 保存现场：保存当前程序状态
  2. 识别中断源：确定中断来源
  3. 执行服务程序：执行相应的中断服务
  4. 恢复现场：恢复程序执行状态

特点

优点 缺点
CPU效率高 硬件复杂
实时性好 中断处理开销
适合低速设备 可能丢失中断

3. 直接存储器访问（DMA）方式

工作原理

  • DMA请求：设备向DMA控制器发送请求
  • 总线控制权：DMA控制器获得总线控制权
  • 直接传输：直接在设备和内存间传输数据
  • 传输完成：传输完成后通知CPU

DMA传输过程

  1. 初始化：CPU设置DMA参数
  2. DMA请求：设备发出DMA请求
  3. 总线仲裁：DMA控制器获得总线控制权
  4. 数据传输：在设备和内存间传输数据
  5. 传输结束：传输完成后发出中断

特点

优点 缺点
高速传输 硬件复杂
CPU开销小 总线竞争
适合批量传输 需要DMA控制器

4. 通道方式

工作原理

  • 通道程序：CPU为通道编写通道程序
  • 独立执行：通道独立执行通道程序
  • 设备控制：通道直接控制设备操作
  • 结果报告：执行完成后向CPU报告

通道类型

  • 字节多路通道：连接多个低速字符设备
  • 选择通道：连接高速块设备
  • 数组多路通道：连接多个高速块设备

特点

优点 缺点
CPU效率最高 硬件成本最高
系统吞吐量大 控制复杂
适合大型系统 不适合小型系统

中断系统

1. 中断的基本概念

中断的定义

中断是指CPU在执行程序过程中，由于某些紧急事件的发生，暂时中止当前程序的执行，转去处理紧急事件，处理完毕后再返回原程序继续执行的过程。

中断的作用

  • 实时响应：及时响应外部事件
  • 提高效率：避免CPU空转等待
  • 异常处理：处理程序执行中的异常情况

2. 中断的分类

按中断源分类

  • 外部中断：来自CPU外部的中断请求
  • 内部中断：由CPU内部产生的中断

按中断性质分类

  • 可屏蔽中断：可以通过软件屏蔽的中断
  • 不可屏蔽中断：不能通过软件屏蔽的中断

按中断优先级分类

  • 高优先级中断：需要优先处理的中断
  • 低优先级中断：可以延迟处理的中断

3. 中断处理过程

中断请求

  • 请求信号：设备发出中断请求信号
  • 请求保持：保持请求信号直到被响应

中断响应

  • 响应条件：CPU允许中断且优先级足够高
  • 响应动作：CPU发出中断响应信号

中断处理

  • 保存现场：保存当前程序状态
  • 识别中断源：确定中断来源
  • 执行服务程序：执行相应的中断服务程序
  • 恢复现场：恢复程序执行状态

4. 中断向量表

结构

  • 中断向量：存储中断服务程序的入口地址
  • 中断类型码：标识不同类型的中断

作用

  • 快速定位：快速找到中断服务程序
  • 统一管理：集中管理所有中断

直接存储器访问（DMA）

1. DMA的基本概念

定义

直接存储器访问（DMA）是一种数据传输方式，允许外部设备直接与内存交换数据，而不需要CPU的干预。

优势

  • 高速传输：实现高速数据传输
  • CPU减负：减少CPU的负担
  • 批量传输：适合大批量数据传输

2. DMA控制器

组成

  • 地址寄存器：存储内存地址
  • 计数寄存器：记录传输数据量
  • 控制寄存器：控制DMA操作
  • 状态寄存器：记录DMA状态

工作模式

  • 单字节模式：每次传输一个字节
  • 块传输模式：连续传输数据块
  • 请求模式：根据设备请求传输

3. DMA传输过程

初始化阶段

  • 设置参数：CPU设置传输参数
  • 启动DMA：启动DMA控制器

传输阶段

  • 总线请求：DMA控制器申请总线控制权
  • 总线响应：CPU释放总线控制权
  • 数据传输：在设备和内存间传输数据

完成阶段

  • 传输结束：完成指定量的数据传输
  • 中断通知：向CPU发出中断通知
  • 总线释放：DMA控制器释放总线

输入输出系统的性能分析

1. 性能指标

响应时间

从发出I/O请求到完成I/O操作的时间。

吞吐量

单位时间内完成的I/O操作数量。

可靠性

系统正常工作的概率。

2. 性能优化技术

缓冲技术

  • 单缓冲：使用单个缓冲区
  • 双缓冲：使用两个缓冲区交替工作
  • 循环缓冲：使用多个缓冲区循环使用

预读技术

  • 顺序预读：预读后续数据
  • 智能预读：根据访问模式预读

缓存技术

  • 磁盘缓存：缓存磁盘数据
  • 网络缓存：缓存网络数据

3. 性能计算示例

磁盘I/O性能

  • 寻道时间：磁头移动到目标磁道的时间
  • 旋转延迟：等待目标扇区到达磁头下的时间
  • 传输时间：实际读取数据的时间

总I/O时间计算

总I/O时间 = 寻道时间 + 旋转延迟 + 传输时间
