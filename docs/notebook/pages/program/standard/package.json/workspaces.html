workspaces

在一个顶层根包中管理多个本地文件系统中的包。

比如我们现在有如下项目结构： 

- packages
    - core
        - package.json
    - tools
        - package.json
- src
    - index.js
- package.json

在根目录下的package.json内容如下： 

{
    "name": "example",
    "private": true,
    "workspaces": [
        "packages/*"
    ],
}

那么，此时packages中的内容就可以看成一个个npm包了，在src中可以类似： 

const Core = require("@example/core");
// ......
