简介

用于构建具有大语言模型（LLM）的有状态、多参与者应用程序的 JavaScript/TypeScript 库。它是 LangChain 团队开发的框架，专门用于创建智能体（Agent）和多智能体工作流。与基于有向无环图（DAG）的传统解决方案不同，LangGraph.js 支持循环流程，这对于大多数智能体架构至关重要。

核心思想是将应用程序的工作流程建模为图结构，其中节点代表不同的操作或步骤，边代表这些步骤之间的关系或流程。这种图形结构允许创建复杂的、有状态的多参与者应用程序，每个参与者可以独立执行操作，相互通信，并根据前一步骤的结果动态调整。

核心概念

图结构（Graph Structure）

LangGraph.js 将应用程序的工作流程表示为一个有向图，包含两个主要元素：

  • 节点（Nodes）：代表应用程序中的不同操作或步骤，如调用 LLM、执行函数、访问数据库等
  • 边（Edges）：定义节点之间的连接关系，控制执行流程和数据流向

状态管理（State Management）

状态是 LangGraph.js 的核心概念之一，它表示应用程序在任意时间点的快照。状态在图的执行过程中在节点之间传递，并且每个节点在执行后使用其返回值更新此内部状态。这种状态管理机制使得应用程序能够记住之前的交互和上下文信息。

循环和分支（Cycles and Branching）

LangGraph.js 支持循环和条件分支，允许在应用程序中实现复杂的控制流。这使得开发者能够构建需要迭代处理或根据条件动态调整执行路径的应用程序。

持久性（Persistence）

LangGraph.js 内置持久性功能，在图的每个步骤后自动保存状态。这支持错误恢复、人机交互工作流、时间旅行等高级功能，允许在任何点暂停和恢复图执行。

人机交互（Human-in-the-Loop）

LangGraph.js 支持中断图执行以批准或编辑智能体计划的下一步操作。这使得在需要人工监督和干预的场景中，如敏感操作审批、复杂决策支持等，能够无缝集成人类输入。

流式支持（Streaming Support）

LangGraph.js 支持在每个节点产生输出时流式传输输出，包括令牌流式输出，提供更好的用户体验和实时反馈。

解决什么问题？

1. 复杂工作流编排

传统的 LLM 应用程序通常使用线性链式结构，这在处理简单任务时效果很好，但在面对需要复杂决策、动态循环和多路径选择的任务时显得力不从心。LangGraph.js 通过图结构提供了更灵活的工作流编排能力。

2. 状态管理和持久化

在构建多轮对话系统或复杂智能体应用时，管理对话历史和上下文是一个挑战。LangGraph.js 的状态管理机制自动处理这个问题，并提供持久化功能，使应用程序能够在执行中断后恢复。

3. 多智能体协作

LangGraph.js 支持创建多智能体系统，其中多个智能体可以协同工作，共同完成复杂任务。每个智能体可以有自己的专业领域和能力，通过图结构进行协调和通信。

4. 人机协作

在某些场景下，完全自动化的系统可能不够可靠或需要人类监督。LangGraph.js 的人机交互功能允许在关键节点暂停执行，等待人类输入或批准，然后再继续执行。

5. 可观测性和调试

LangGraph.js 提供了对应用程序流程和状态的细粒度控制，这对于创建行为可靠、符合预期的智能体至关重要。开发者可以轻松地跟踪和调试应用程序的执行过程。
