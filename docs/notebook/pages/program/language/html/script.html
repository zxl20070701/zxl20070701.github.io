script标签

script标签用于在HTML文档中嵌入或引用JavaScript代码，是前端开发中最重要的标签之一。通过script标签，我们可以为网页添加交互功能。

属性详解

src属性

src属性用于指定外部JavaScript文件的URL。当设置了src属性时，script标签内部的内容将被忽略。

async属性

async属性是布尔属性，表示脚本应该异步执行。当浏览器遇到带有async属性的script标签时，会立即开始下载脚本，同时在后台继续解析HTML。脚本下载完成后会立即执行，可能会中断HTML解析。

使用场景：适用于独立脚本，不依赖其他脚本，也不被其他脚本依赖的情况。

defer属性

defer属性也是布尔属性，表示脚本应该延迟执行。带有defer属性的脚本会等到HTML解析完成后再执行，并且按照它们在文档中出现的顺序执行。

使用场景：适用于需要等待DOM加载完成的脚本，或者需要按照顺序执行的脚本。

type属性

type属性指定脚本的MIME类型。对于JavaScript，通常是"text/javascript"或"application/javascript"。在现代HTML中，这个属性可以省略。

charset属性

charset属性指定外部脚本的字符编码。这个属性很少使用，因为大多数服务器都会正确设置HTTP响应头。

crossorigin属性

crossorigin属性用于配置CORS（跨域资源共享）请求。当脚本来自不同的源时，这个属性很有用。

integrity属性

integrity属性允许浏览器检查获取的资源是否被篡改，提供了额外的安全性。通常与crossorigin属性一起使用。

加载策略对比

策略 加载时机 执行时机 执行顺序 适用场景
普通script 立即加载 立即执行，阻塞HTML解析 按出现顺序 简单页面，无特殊要求
async 异步加载 加载完成后立即执行 不确定，取决于加载完成顺序 独立脚本，如统计代码
defer 异步加载 HTML解析完成后执行 按出现顺序 依赖DOM的脚本

最佳实践

放置位置

传统上，script标签通常放在body结束标签之前，这样可以确保DOM已经完全加载。但是，现代做法更倾向于将script标签放在head中，并使用defer属性。

性能优化

  • 对于关键脚本，使用普通的script标签并放在页面底部
  • 对于非关键脚本，使用async或defer属性
  • 对于第三方脚本（如分析工具），优先使用async属性
  • 对于依赖DOM的脚本，使用defer属性

注意事项

  • 内联脚本不能使用async或defer属性
  • async和defer属性仅对外部脚本有效
  • 不要在同一个script标签上同时使用async和defer属性
  • 动态创建的script标签默认是async的
