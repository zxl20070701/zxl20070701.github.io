weex模块

什么是Weex模块

Weex模块是Weex提供的原生功能接口，通过模块可以调用设备的原生能力，如网络请求、存储、设备信息等。

常用内置模块

  • modal: 弹窗模块，提供alert、confirm、prompt等弹窗功能
  • navigator: 导航模块，用于页面跳转
  • storage: 存储模块，提供键值对存储功能
  • stream: 网络请求模块，用于发送HTTP请求
  • clipboard: 剪贴板模块，用于复制粘贴文本
  • picker: 选择器模块，提供日期、时间、颜色等选择功能
  • deviceInfo: 设备信息模块，获取设备相关信息

模块使用方式

// 引入模块
const modal = weex.requireModule('modal');
const navigator = weex.requireModule('navigator');
const storage = weex.requireModule('storage');

// 使用modal模块
modal.toast({
  message: '这是一个提示',
  duration: 2
});

modal.alert({
  message: '这是一个警告框',
  okTitle: '确定'
}, function() {
  console.log('点击了确定按钮');
});

// 使用navigator模块
navigator.push({
  url: 'next-page.js',
  animated: 'true'
});

// 使用storage模块
storage.setItem('key', 'value', function(e) {
  if (e.result === 'success') {
    console.log('存储成功');
  }
});

storage.getItem('key', function(e) {
  if (e.result === 'success') {
    console.log('获取到的值：', e.data);
  }
});

自定义模块

除了内置模块，Weex还支持自定义模块。开发者可以在原生端（Android/iOS）实现自定义模块，然后在JS端调用。

模块调用注意事项

  • 模块调用是异步的，需要传入回调函数处理结果
  • 不同平台可能支持的模块不同，需要做好兼容性处理
  • 模块调用失败时，需要做好错误处理
