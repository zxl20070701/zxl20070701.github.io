weex开发问题
  1. weex默认的屏幕宽度为750，只支持px,不支持百分比；
  2. 仅支持单class选择器，不支持样式继承；
  3. weex只支持四种伪类：active、focus、disabled、enabled；
  4. 默认为flex布局，纵向布局，子元素拉伸填充横向；
  5. css属性不支持简写，属性名要写全；
  6. background-image四不能使用图片资源，图片只支持iamge标签；
  7. 文本必须使用text标签，不支持一行文本多种样式，如果需要的话可使用富文本；
  8. Android 网不支持box-shadow这个属性；
  9. 不支持display:none，不支持v-show，只能用v-if；
  10. 虽然使用的是vue语法，但是不支持v-html。
  11. weex生命周期与vue不完全相同，部分生命周期钩子不可用或行为有差异。
  12. 组件库有限，无法直接使用web端丰富的UI组件库。
  13. 事件处理不支持事件冒泡和捕获阶段，事件委托实现困难。
  14. 不支持z-index属性，层级控制只能通过DOM顺序实现。
  15. 调试困难，无法像web开发一样使用浏览器开发者工具。
  16. 性能优化复杂，需要手动处理内存泄漏和性能瓶颈。
  17. Android和iOS平台渲染效果可能存在细微差异，需要针对性适配。
  18. 网络请求必须使用特定的stream模块，不能直接使用fetch或axios。
  19. 本地存储能力有限，无法像web一样使用localStorage或sessionStorage。
  20. 插件生态相对薄弱，需要自行实现很多原生功能。
