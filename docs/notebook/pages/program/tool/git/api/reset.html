撤销

回退本地文件

将特定的filename文件回滚到hashname提交。 

git checkout hashname filename

删除本地修改

用暂存区的所有文件直接覆盖本地文件，不给用户任何确认的机会，可以理解放弃本地修改。 

git checkout .

只会放弃已跟踪文件的修改，如果是未跟踪的文件和目录需要删除，可以： 

git clean -df

撤销add

把add添加的文件filename从缓冲区删除，恢复到没有add前（不会丢失文件）。 

git reset -- filename

删除提交

恢复到hashname指定的提交时，不保留修改。 

git reset --hard hashname

温馨提示：比如你想删除本次提交，就像什么也没有发生一样，上面的hashname使用这次提交的前一个即可。

撤销提交

恢复到hashname指定的提交时，保留修改。 

git reset --soft hashname

温馨提示：比如你想撤销本次提交，恢复到add状态，上面的hashname使用这次提交的前一个即可。

简化语句

比如我们想撤销上一次提交，可以简化的写成： 

git reset HEAD~1

其中数字1表示撤销最近的一次提交，可以修改数字来调整撤销的提交次数（可以不传，默认为1），比如撤销最近的五次提交： 

git reset HEAD~5
