什么是HTML？

这是一种用来组织网页内容的标记语言（或数据格式），你可以通过它来配置网页的基本结构（配置和内容），后续的所有操作几乎全部是基于此进行的。

页面结构 

<!DOCTYPE html>
<html lang="zh-cn">

<!-- 页面配置 -->
<head>
    <meta charset="UTF-8">
    <title>学习HTML</title>
</head>

<!-- 页面内容 -->
<body>
    今天，我们来一起学习HTML!
</body>

</html>

上面是页面基本结构，在 head标签 中对页面进行配置，在 body标签 中编辑显示内容。我们建议你记住上面这段代码，这有助于你后续的学习！

标签分类和说明

从上面的基本结构你就可以看到，不同的标签有不同的功能划分，下面，按照我们的理解对标签进行分类并举一些常用的例子。

配置标签

比如开头说明的标签中大部分都是配置标签，这一部分目前对你不是特别重要，你需要记住下面几个常用的即可：

设置网页编码： 

    <meta charset="UTF-8">

设置网页标题： 

    <title>标题</title>

温馨提示：一定要注意这些标签的位置，下同！

语言型标签

也就是可以内嵌别的语言的标签，最典型的就是 sytle标签 和 script标签 了，一个和css相关，一个和js相关。

sytle标签

css用于控制页面的样子（当然是在原有的HTML基础上进行改造，关于css知识后续可以去对应章节学习，这里不用太在意），代码需要写在 sytle标签 规定的区域。比如我们想让所有的div标签里面的文字变成红色： 

<style>
div{
    color:red;
}
</style>

当然，我们也可以把style标签之间的内容写到一个独立的文件中，比如 style.css ，然后引入： 

    <link rel="stylesheet" href="style.css">

script标签

js用于提供页面和用户的交互，代码需要写在 script标签 规定的区域（关于js部分的代码，你可以先不用太关心）。假设我们现在希望在控制台打印一个语句，代码如下： 

<script>
    console.log('你好呀，这里是控制台~');
</script>

同样的，我们也可以把script标签之间的内容写入到一个独立的文件中，比如 script.js ，然后引入： 

    <script src="script.js"></script>

内容或布局元素

这一类标签虽然很多，其实都是一样的，差异主要是由于不同标签的默认样式（比如上面说的文字颜色 color : red 就是样式）不同导致的，还有就是部分标签有一定的特殊意义，我们来分别说明。

盒子模型

在正式开始说明之前，我们有必要简单的说明一下盒子模型。

可以看见，一个元素相当于一个矩形，最里面是内容（content），最外边是边界（border)，内容和边界之间存在一个间距（padding）,而盒子之间也存在一个距离（margin)。

盒子还会有两个属性宽和高，来标记这个盒子多大，可以有多种计算方式，你应该记住下面两种计算方式：

  • 标准盒子模型 ：这种模型是大部分浏览器默认的模型，计算宽高的时候，只考虑到content的大小。
  • IE盒子模型 ：计算宽高的时候，会把除了margin外的都考虑进去。

文档流

你可以把页面内容中的标签看成一个个盒子（盒子模型-+），默认情况下，按照从左到右，从上往下的顺序排列。而这种排列规则，就叫文档流。

下面要说明的这部分标签，他们在文档流中的表现，会有一些差异，记住这些差异是有益的。

不同标签在文档流中占据的位置的大小计算规则不一样，在默认情况下，span占据的宽取决于里面的内容，而div无论里面内容多少，都是占满一行。

我们称span这类元素为 行内元素（inline） ，而称div这类为 块级元素（block） 。当然，他们的区别不只是上面提到的这些，先记住这一点就可以了，后续我们会慢慢补充的。

所以，p标签和div相比，就是多了外边距（div的外边距大小默认为0）。

再强调一下，不同标签有很多默认属性（后续会说明如何修改），这一类都可以归结到文档流中最终呈现的效果不同的理解上去。

而在这里，h1~h6标签更重要的意义体现在其本身的意义，比如一篇文章的题目用h1，每个段落的标题用h2等，虽然使用别的标签可能最终在页面上显示的效果是一样的，可是使用h1~h6更有语义，别人看一眼（看源码的时候）就知道是一个标题等。

除此之外，还有很多，比如导航nav、页脚footer、页眉header等，不用一下子都记住，用的多了就记住了。

同样的，除了列表，还有表格table、表单form（这个可能更特殊）等，这类标签包裹的数据之间都具有一定的联系。你不需要一下子记住那么多，慢慢积累就可以了。

小结

我们举这么多例子并不是希望大家都记住这些标签，当然，更不可能让你去记住所有的标签，主要的目的是让你去理解标签。

不同的标签肩负着不同的使命，所以以后在遇到一个未知的标签的时候，首先需要去理解，这个标签存在的意义是什么，然后，你就知道，什么时候应该用这个标签，以及怎么用。
